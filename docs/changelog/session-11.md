## 2026-03-30 — Session 11: Claude API Packing List Generator

### First AI Feature

### Created
- `lib/claude.ts` — Claude API client using `@anthropic-ai/sdk`. Prompt engineering for packing list generation: sends trip details, full gear inventory (by category with IDs), weather forecast, and alerts to Claude Sonnet. Returns structured JSON with categorized items, inventory links, reasons, and trip tips.
- `app/api/packing-list/route.ts` — POST endpoint. Fetches trip + relations, owned gear inventory, and weather (if location has GPS and within forecast range). Passes everything to Claude. Full error handling.
- `components/PackingList.tsx` — Full packing list UI component:
  - "Generate with Claude" CTA button (amber, matches style guide)
  - Loading state with spinner + skeleton rows
  - Error state with retry button
  - Categorized items with emoji headers matching gear categories
  - Custom amber checkboxes with line-through on pack
  - Progress bar (packed / total)
  - "add" badge on items not in inventory
  - Inline "Add item" per category for manual additions
  - Trip tips section from Claude
  - "✦ Generated by Claude · Edit freely" attribution
  - Full dark mode support

### Changed
- `components/TripsClient.tsx` — Added PackingList component below WeatherCard on upcoming trip cards
- `package.json` — Added `@anthropic-ai/sdk` dependency

### Key Decisions
- **Claude Sonnet for generation** — fast enough for interactive use, smart enough for context-aware packing
- **Prompt includes gear IDs** — Claude references exact inventory items so the UI can link back to gear details later
- **Weather-aware** — if forecast is available, Claude adjusts recommendations (rain gear, warm layers, sun protection)
- **Editable after generation** — users can check items, add custom items, and regenerate freely
- **No persistence yet** — packing list lives in component state. Will save to PackingItem model when trip prep flow is built (task #5)
