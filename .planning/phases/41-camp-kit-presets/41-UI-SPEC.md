> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |

## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, checkbox-to-label gap (`gap-1`) |

| lg | 24px | Section breaks inside panels (`space-y-6`) |

Note: Gear list row height inside KitStackPanel is an implementation-level detail — match the existing KitPresetsPanel visual rhythm at implementation time. No specific px value is declared in this contract.

## Color

| Role | Light value | Dark value | Usage |
|------|------------|------------|-------|
| Dominant (60%) | `#fafaf9` stone-50 (`--color-bg-primary`) | `#0c0a09` stone-950 | Page background, panel body background |
| Secondary (30%) | `#ffffff` white (`--color-bg-surface`) | `#1c1917` stone-900 | Panel white surface, card border areas |
| Accent (10%) | `#d97706` amber-600 (`--color-accent`) | `#f59e0b` amber-500 | Reserved for specific elements listed below |
| Destructive | `#dc2626` red-600 (`--color-danger`) | `#ef4444` red-500 | Kit delete button only |

Accent (`amber-600` light / `amber-500` dark) is reserved for:
1. Primary action buttons: "Apply Kits" button, "Create Kit" button, "Save as Kit" confirm button
2. Checkbox `accent-amber-600` on gear picker checkboxes and kit multi-select checkboxes
3. Focus ring (`focus:ring-amber-400`) on all text inputs
4. Border hover on kit selection items (`hover:border-amber-400`)

Accent is NOT used for: the "Ask Claude to review" button (see interaction contract below), applied-kit chips, secondary text, or icons in neutral state.

### "Ask Claude to review" button — amber ghost variant
Source: CONTEXT.md `## Claude's Discretion` — visual treatment deferred to researcher.
Decision: Use an amber ghost/outline style, not the filled amber primary style. This distinguishes the optional AI step from the primary "Apply Kits" action and signals lower commitment.
Exact classes: `border border-amber-500 dark:border-amber-400 text-amber-700 dark:text-amber-400 hover:bg-amber-50 dark:hover:bg-amber-900/30 rounded-xl px-4 py-2.5 font-semibold text-sm transition-colors`

### Applied-kit chips
Applied-kit chips (e.g., "Weekend Warrior ✕") use the secondary surface with a stone border — no accent color.
Exact classes: `bg-stone-100 dark:bg-stone-800 border border-stone-200 dark:border-stone-700 text-stone-800 dark:text-stone-200 rounded-full px-3 py-1 text-sm flex items-center gap-1.5`
Remove (✕) button within chip: `text-stone-400 hover:text-red-500 dark:hover:text-red-400 transition-colors` with `aria-label="Remove {kit name}"` (substitute the actual kit name at render time).

### Success state
Inline success messages use `text-emerald-600 dark:text-emerald-400 text-sm` — matches existing codebase pattern (RESEARCH.md verified).

## Component Inventory

New components and interaction states needed for this phase.

### KitStackPanel (new component — `components/KitStackPanel.tsx`)

A slide-up modal that replaces the existing single-apply kit dropdown in `PackingList.tsx`. Contains:

1. **Header** — "Apply Kit Presets" title + `✕` close button with `aria-label="Close"`. Same sticky header pattern as KitPresetsPanel.
2. **Kit checkbox list** — All saved kits as checkboxes. Multi-select. Each row shows kit name + item count.
   - Item count shown as secondary text: "8 items" in `text-xs text-stone-400 dark:text-stone-500`
   - Source: CONTEXT.md `## Claude's Discretion` resolved — show item counts alongside names (more informative than names alone, low cost)
3. **Applied-kits tracker** — Displayed below the kit list when 1+ kits are selected. Label: "Will apply:" then chip list.
4. **"Apply Kits" primary button** — amber filled, full width, disabled when no kits selected.
5. **Empty state** — when no kits exist: "No kit presets yet. Create one from the Gear page."

Slide-up animation: use existing `.animate-slide-up` class from `globals.css`.
Backdrop: `fixed inset-0 bg-black/50 dark:bg-black/70 flex items-end sm:items-center justify-center animate-fade-in` — matches KitPresetsPanel exactly.
Panel container: `bg-white dark:bg-stone-900 rounded-t-2xl sm:rounded-2xl w-full max-w-lg max-h-[90vh] overflow-y-auto shadow-xl animate-slide-up`

### Applied-kits tracker (inline in PackingList.tsx — NOT a separate component)

Shown below the kit picker trigger after at least one kit has been applied. Renders horizontally scrollable row of chips.

Layout: `flex flex-wrap gap-2 items-center` with a "Applied:" label in `text-xs text-stone-500 dark:text-stone-400 font-semibold` prefix.

Each chip: kit name + `✕` remove button with `aria-label="Remove {kit name}"`. Removing a kit triggers unapply logic.

Loading state during unapply: spinner replaces `✕` on the specific chip being removed; chip turns `opacity-50`. Other chips remain interactive.

### "Save as Kit" inline UI (inside PackingList.tsx — after packing list is generated)

A compact inline form that appears below the generated packing list. Not a full slide-up modal.
Source: RESEARCH.md Open Question 2 — resolved as inline input, not KitPresetsPanel.

Layout:
- Heading: `text-sm font-semibold text-stone-700 dark:text-stone-300` — "Save this list as a kit preset"
- Secondary: `text-xs text-stone-500 dark:text-stone-400` — "Saves {N} gear items from your inventory"
- Input: same `px-3 py-2.5 rounded-lg border border-stone-300 dark:border-stone-600 bg-white dark:bg-stone-800` pattern
- "Save Kit" button: amber filled, same style as primary button

Collapsed state: a single `text-sm text-amber-700 dark:text-amber-400 underline` text link — "Save as kit preset" — that expands the form on tap. Keeps the UI light until the user wants the action.

### "Ask Claude to review" button (inline in PackingList.tsx)

Appears in PackingList.tsx after 1+ kits have been applied (not before). Uses amber ghost variant defined in Color section.

Loading state during review: `<Loader2 className="w-4 h-4 animate-spin mr-2" />` inside the button, text becomes "Reviewing..." — same pattern as existing "Generating..." in handleGenerate.

Review result display: inline `div` below the button, rendered as a bulleted list. Container: `bg-amber-50 dark:bg-amber-900/20 border border-amber-200 dark:border-amber-800 rounded-xl p-4 mt-3`. Text: `text-sm text-stone-800 dark:text-stone-100`. Bullets from Claude rendered as `<ul className="list-disc list-inside space-y-1 text-sm">`.

## Interaction States

All interactive elements must implement these states:

| State | Visual |
|-------|--------|
| Default | As specified above |
| Hover | Tailwind `hover:` variant (bg, border, text shift — see color per element) |
| Focus | `focus:outline-none focus:ring-2 focus:ring-amber-400` on all inputs |
| Loading | Opacity 50 (`disabled:opacity-50`) + spinner on button where applicable |
| Disabled | `disabled:opacity-50 disabled:cursor-not-allowed` |
| Error | Inline `text-sm text-red-600 dark:text-red-400` below the failing element |
| Success | Inline `text-sm text-emerald-600 dark:text-emerald-400` below the action |

### Destructive: Delete Kit
Element: "Delete" button on kit row in KitPresetsPanel (existing, no change needed)
Confirmation approach: No modal required for kit delete — kits can be recreated easily. The delete button is already styled in red (`text-red-600`). This matches the existing pattern in KitPresetsPanel.tsx (direct delete, no confirm dialog). No change.

### Destructive: Remove applied kit (chip ✕)
This is NOT destructive enough to require a confirmation dialog. Removing a kit from the applied-kits tracker removes its PackingItems, but the user is actively in the packing session. The remove is instant with a 3-second success message. If they remove by mistake they can re-apply.

## Layout Hierarchy (PackingList.tsx kit section)

The kit section in PackingList.tsx has this visual hierarchy after Phase 41:

```
[ Kit Presets section ]
  "Use Kit Presets" button (amber ghost, full width or inline)
  |
  Applied: [Weekend Warrior ✕] [Dog Trip ✕]   ← appears after kits applied
  |
  "Ask Claude to review" button (amber ghost)  ← appears after kits applied
  |
  [ Review result card ]                       ← appears after review runs

[ Generate with Claude section ]               ← existing, unchanged
  "Generate Packing List" button
  |
  [ Packing list result ]
  |
  "Save as kit preset" link → [ Save form ]    ← appears after list exists
```

These two sections are visually separated. Do not merge them. Source: CONTEXT.md D-04, RESEARCH.md anti-patterns.
