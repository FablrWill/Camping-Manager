# Phase 13, Plan 01: Fix High-Severity Review Findings

## Objective

Fix all HIGH-severity issues from the Gemini cross-AI review so the codebase shipping to production has been vetted and hardened by two AI models.

## Context

- @.planning/phases/12-fix-build-clean-house/GEMINI-REVIEW.md — Full Gemini review report

## HIGH Issues to Fix

6 HIGH-severity findings from the Gemini review:

1. **Path Traversal in Photo Import** — `app/api/import/photos/route.ts` line 53: `photo.imagePath` concatenated with `takeoutRoot` without sanitization. Fix: resolve path and verify it stays within `takeoutRoot`.
2. **Path Traversal in Photo Delete** — `app/api/photos/[id]/route.ts` line 19: `photo.imagePath` joined with `public/` without sanitization. Fix: validate path stays within `/public/photos/`.
3. **Unvalidated JSON.parse on LLM Output (voice extract)** — `lib/voice/extract.ts` line 30: raw `JSON.parse(text)` with no schema validation. Fix: use `parseClaudeJSON` with `InsightPayloadSchema`.
4. **Unvalidated JSON.parse on LLM Output (memory)** — `lib/agent/memory.ts` line 88: raw `JSON.parse(text)` on LLM output. Fix: wrap in try/catch with Zod schema validation.
5. **Unvalidated JSON.parse on Chat UI** — `components/ChatBubble.tsx` lines 20, 33: JSON.parse without schema validation. **Already has try/catch** — existing code wraps in try/catch. Add Zod validation.
6. **XSS in Leaflet Popups** — `components/SpotMap.tsx` lines 200, 225: raw DB strings interpolated into popup HTML. Fix: add HTML escaping helper.

## Tasks

### Task 1: Fix path traversal in photo import and delete (type=auto)

**Files:**
- `app/api/import/photos/route.ts`
- `app/api/photos/[id]/route.ts`

**Changes:**
- In `import/photos/route.ts`: after computing `sourcePath = join(takeoutRoot, photo.imagePath)`, resolve both `takeoutRoot` and `sourcePath` with `path.resolve()` and verify `sourcePath` starts with the resolved `takeoutRoot`. If not, skip the photo with an error entry.
- In `photos/[id]/route.ts`: after computing `filePath = path.join(cwd, 'public', photo.imagePath)`, verify it starts with `path.join(cwd, 'public', 'photos')`. If not, skip the delete (treat as non-fatal — the record still gets deleted from DB).

**Done:** Both files sanitize file paths before use. No path traversal possible.

---

### Task 2: Fix unvalidated JSON.parse on LLM output (voice extract + memory) (type=auto)

**Files:**
- `lib/voice/extract.ts`
- `lib/agent/memory.ts`
- `lib/parse-claude.ts` (add InsightPayloadSchema if not already present)

**Changes:**
- `lib/voice/extract.ts`: Replace `JSON.parse(text) as InsightPayload` with `parseClaudeJSON(text, InsightPayloadSchema)`. On failure, throw a typed error with the validation message so the API route returns 500.
- `lib/agent/memory.ts`: The function already has a top-level try/catch. Wrap the `JSON.parse(text)` in an inline try/catch and add Zod validation against a `MemoryArraySchema` (`z.array(z.object({ key: z.string(), value: z.string() }))`). On parse failure, return early (memory is fire-and-forget).
- `lib/parse-claude.ts`: Add `InsightPayloadSchema` (matching `InsightPayload` from `lib/voice/types.ts`) if not already defined there. Export it.

**Done:** LLM output is validated before use; malformed responses return clear errors instead of crashing.

---

### Task 3: Fix XSS in Leaflet popups (type=auto)

**Files:**
- `components/SpotMap.tsx`

**Changes:**
- Add a local `escHtml(s: string): string` utility at the top of the component file that escapes `&`, `<`, `>`, `"`, `'`.
- Apply `escHtml()` to all user-/DB-sourced strings in popup HTML templates:
  - Photo popups: `photo.title`, `photo.imagePath` (src/alt), `photo.locationDescription`, `photo.googleUrl`
  - Location popups: `loc.name`, `loc.description`
  - Place visit popups: `pv.name`, `pv.address`
- Do NOT escape system-generated values (ratings stars, type labels computed from enum, durations computed numerically, `photo.id` used as data attribute).

**Done:** No DB-sourced strings are interpolated into popup HTML without escaping.
