    **Step 1: Build the review prompt file**

    Create a temporary file at `/tmp/gemini-review-prompt.txt` with this content:

    ```
    You are reviewing the Outland OS codebase — a personal car camping assistant built as a Next.js 16 mobile-first web app with SQLite/Prisma, Tailwind CSS, and Claude AI integration. Single user, no auth. About to be deployed to production on a Mac mini via PM2.

    Review the following codebase files for:
    1. Security vulnerabilities (injection, XSS, CSRF, exposed secrets, unsafe data handling)
    2. Production readiness issues (error handling gaps, missing validation, crash risks)
    3. Performance concerns (N+1 queries, memory leaks, unbounded operations)
    4. Code quality issues (dead code, inconsistencies, missing error boundaries)
    5. Architecture concerns (coupling, scalability bottlenecks, data integrity risks)

    For each finding, provide:
    - **File**: exact file path
    - **Line(s)**: approximate line numbers
    - **Issue**: concise description
    - **Severity**: CRITICAL / HIGH / MEDIUM / LOW
    - **Suggested fix**: specific remediation

    Group findings by severity. Start with CRITICAL, then HIGH, MEDIUM, LOW.

    If no findings exist for a severity level, state "None found."

    CODEBASE FILES FOLLOW:
    ```

    **Step 2: Concatenate codebase files**

    Append the contents of all source files to the prompt. Include:
    - All `app/**/*.ts` and `app/**/*.tsx` files (pages, API routes, layouts)
    - All `components/**/*.tsx` files
    - All `lib/**/*.ts` files
    - `prisma/schema.prisma`
    - `public/sw.js`
    - `next.config.ts`
    - `middleware.ts` (if exists)

    Use this shell pattern:
    ```bash
    for f in $(find app components lib -name '*.ts' -o -name '*.tsx' | sort) prisma/schema.prisma public/sw.js next.config.ts; do
      echo "--- FILE: $f ---"
      cat "$f"
      echo ""
    done >> /tmp/gemini-review-prompt.txt
    ```

    **Step 3: Run Gemini review**

    ```bash
    npx gemini -p "$(cat /tmp/gemini-review-prompt.txt)" --yolo 2>&1
    ```

    If the prompt is too large for a single call (Gemini may truncate), split into batches:
    - Batch 1: app/ directory files
    - Batch 2: components/ directory files
    - Batch 3: lib/ + prisma/ + config files
    Then run a final synthesis prompt asking Gemini to merge findings from all batches.

    **Step 4: Save and format the report**

    Save Gemini's output to `.planning/phases/12-fix-build-clean-house/GEMINI-REVIEW.md` with this structure:

    ```markdown
    ## Summary

    - Critical: {N}, High: {N}, Medium: {N}, Low: {N}

    ```
