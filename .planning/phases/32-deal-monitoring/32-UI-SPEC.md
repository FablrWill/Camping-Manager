## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding (`px-2 py-0.5`) |
| sm | 8px | Compact element spacing (`p-2`, `gap-2`) — price range label + deal badge |
| md | 16px | Default element spacing (`p-4`) — Deals tab content padding, card padding |
| lg | 24px | Section padding (`space-y-6`) — tab section separators |
| xl | 32px | Layout gaps (`p-8`) — modal content area |
| 2xl | 48px | Major section breaks — dashboard card top margin |
| 3xl | 64px | Page-level spacing — not used in this phase |

Exceptions:
- Touch targets for "Check Price" and "Re-check" buttons: minimum 44px height on mobile (`min-h-[44px]`), consistent with existing Button component usage in modals.

Source: Observed patterns in `GearResearchTab.tsx`, `Badge.tsx`, `Button.tsx`. Standard 8-point grid applies.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 12px (`text-xs`) | 500 (medium) | 1.4 |
| Heading | 16px (`text-base`) | 600 (semibold) | 1.2 |
| Display | 20px (`text-xl`) | 700 (bold) | 1.2 |

Notes:
- Deals tab content follows `GearResearchTab.tsx` patterns: item name at `text-sm font-semibold`, metadata (checkedAt, price range) at `text-xs`.
- Target price and found price range displayed at `text-sm font-semibold` to give equal visual weight.
- Staleness disclaimer: `text-xs` weight 400, `--color-text-secondary` (stone-500 light / stone-400 dark).
- Dashboard "Deals (N)" card heading: `text-base font-semibold` matching "Upgrade Opportunities" heading pattern.

Source: `GearResearchTab.tsx` lines 36-55, `Badge.tsx` (`text-xs font-medium`).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg-primary` (#fafaf9 light / #0c0a09 dark) | Page background, modal backdrop |
| Secondary (30%) | `--color-bg-surface` (#ffffff light / #1c1917 dark) + `--color-bg-surface-alt` (#f5f5f4 light / #292524 dark) | Cards in Deals tab, tab content area, dashboard card |
| Accent (10%) | `--color-accent` (#d97706 amber-600 light / #f59e0b amber-500 dark) | "Check Price" and "Re-check" primary button backgrounds only |
| Destructive | `--color-danger` (#dc2626 red-600 light / #ef4444 red-500 dark) | Stale warning icon color only |

Accent reserved for:
- "Check Price" button (primary CTA inside Deals tab)
- "Re-check" button (secondary action after result is shown)

NOT used for: deal status badge, target price label, price range display, tab button active state (which uses stone-900/stone-50 text contrast).

Deal status badge colors (semantic, not accent):
- Active deal ("Deal!"): `bg-emerald-50 text-emerald-700 dark:bg-emerald-950/30 dark:text-emerald-300` — matches Badge `new` variant.
- No deal yet: `bg-stone-100 text-stone-600 dark:bg-stone-800 dark:text-stone-400` — matches Badge `default` variant.
- Stale warning: `--color-warning` (#d97706 amber-600) icon + `text-amber-700 dark:text-amber-300` text — matches `GearResearchTab` stale pattern.

Target price badge on wishlist gear cards:
- Target price label: `bg-stone-100 text-stone-600 dark:bg-stone-800 dark:text-stone-400` (Badge `default` variant), prefix with Tag icon.
- Green "Deal!" indicator on card: `bg-emerald-50 text-emerald-700 dark:bg-emerald-950/30 dark:text-emerald-300` (Badge `new` variant), prefix with CheckCircle icon.

Source: `globals.css` custom properties, `Badge.tsx` variant styles, `GearResearchTab.tsx` VERDICT_CONFIG color classes.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Deals tab label | "Deals" |
| Primary CTA (no prior check) | "Check Price" |
| Re-check CTA (result exists) | "Re-check" |
| Loading state message | "Checking prices…" |
| Empty state heading (no target price set) | "No target price set" |
| Empty state body (no target price set) | "Add a target price in the gear edit form to start monitoring deals." |
| Empty state heading (target price set, no check run) | "No price check yet" |
| Empty state body (target price set, no check run) | "Check current prices using Claude's training data. Prices may be outdated." |
| Deal status — active deal | "Deal!" |
| Deal status — no deal | "No deal yet" |
| Staleness warning | "Price data is over 30 days old — consider re-checking." |
| Staleness disclaimer (always visible) | "Prices based on Claude's training data — may be outdated." |
| Error state | "Price check failed. Check your connection and try again." |
| Dashboard card heading (deals exist) | "Deals (N)" |
| Dashboard card empty (no active deals) | Card is hidden — renders nothing when `initialDeals` is empty. |
| Dashboard deal entry format | "{Item Name} — Target: \${targetPrice} / Found: {foundPriceRange}" |
| Target price field label (GearForm) | "Target price" |
| Target price field placeholder | "e.g. 89" |

Notes:
- "Deals (N)" uses the live count of active deals as N — e.g., "Deals (3)".
- No destructive actions in this phase — no confirmations required.
- The staleness disclaimer is always visible below any price check result, not just when stale. The stale warning is an additional banner shown only when `isStale` is true.

Source: D-03, D-04, D-06, D-08, D-09 from CONTEXT.md. Wording follows GearResearchTab patterns ("Research data is over 90 days old" → adapted to 30 days).

Only show "Deal!" badge when `isAtOrBelowTarget === true`. Always show target price badge when `targetPrice` is set.

## Interaction States

### "Check Price" Button

| State | Appearance |
|-------|------------|
| Default | Amber accent background, white text, 44px min-height |
| Loading | Spinner (RefreshCw icon, animate-spin) + "Checking prices…" text, disabled |
| Error | Button re-enabled; error message displayed inline below button in `--color-danger` text |

### Stale Warning Banner

| Condition | Appearance |
|-----------|------------|
| `isStale === false` | Not rendered |
| `isStale === true` | Amber background `bg-amber-50 dark:bg-amber-950/30`, AlertTriangle icon in amber, warning text |

### Deals Tab Button Visibility

| Item type | Tab visible |
|-----------|-------------|
| Owned gear (`isWishlist === false`) | Tab button not rendered |
| Wishlist gear (`isWishlist === true`) | Tab button rendered as third tab |
