Create `app/api/chat/route.ts` — POST-only streaming endpoint.

**Addresses review concern #3 (HIGH):** BetaToolRunner has a confirmed `max_iterations` parameter built into the SDK. Set it to 8. No manual iteration tracking or spike needed.

```typescript
import { NextRequest } from 'next/server'
import Anthropic from '@anthropic-ai/sdk'
import { AGENT_TOOLS } from '@/lib/agent/tools'
import { CAMPING_EXPERT_SYSTEM_PROMPT } from '@/lib/agent/system-prompt'
import { buildContextWindow, extractAndSaveMemory } from '@/lib/agent/memory'
import { prisma } from '@/lib/db'

export const dynamic = 'force-dynamic'

const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })
```

**POST handler logic:**

1. Parse request body: `{ conversationId: string | null, userMessage: string, pageContext?: string }`
2. Validate `userMessage` is non-empty, return 400 if missing
3. Create or reuse conversation:
   - If `conversationId` is null, create new: `prisma.conversation.create({ data: { title: userMessage.slice(0, 100) } })`
   - If provided, verify it exists with `prisma.conversation.findUnique`. If not found, create new.
4. Build context window BEFORE saving user message (avoids duplicate — buildContextWindow loads from DB): `const contextMessages = await buildContextWindow(activeConversationId, pageContext)`
5. Append the new user message: `contextMessages.push({ role: 'user', content: userMessage })`
6. Save user message to DB (after building context, so it's not loaded twice): `prisma.message.create({ data: { conversationId: activeConversationId, role: 'user', content: userMessage } })`

**Streaming with BetaToolRunner:**

Create a `ReadableStream` and return it as the response:

```typescript
const encoder = new TextEncoder()
const stream = new ReadableStream({
  async start(controller) {
    const send = (event: string, data: unknown) => {
      controller.enqueue(encoder.encode(`event: ${event}\ndata: ${JSON.stringify(data)}\n\n`))
    }

    try {
      let fullAssistantText = ''
      const toolCallNames: string[] = []

      // BetaToolRunner with confirmed max_iterations parameter
      const runner = client.beta.messages.toolRunner({
        model: 'claude-sonnet-4-20250514',
        max_tokens: 2048,
        system: CAMPING_EXPERT_SYSTEM_PROMPT,
        tools: AGENT_TOOLS,
        messages: contextMessages,
        stream: true,
        max_iterations: 8,  // SDK-native iteration cap (CHAT-04, review concern #3)
      })

      // Iterate over turns — each turn is a BetaMessageStream when stream: true
      for await (const turn of runner) {
        // Attach event listeners to each stream turn
        turn.on('text', (delta: string) => {
          fullAssistantText += delta
          send('text_delta', { delta })
        })

        turn.on('contentBlock', (block: any) => {
          if (block.type === 'tool_use' && block.name) {
            send('tool_activity', { toolName: block.name })
            toolCallNames.push(block.name)
          }
        })

        // Wait for this turn to complete before processing next
        await turn.finalMessage()
      }

      // Save assistant message to DB
      await prisma.message.create({
        data: {
          conversationId: activeConversationId,
          role: 'assistant',
          content: fullAssistantText,
          toolCallsJson: toolCallNames.length > 0 ? JSON.stringify(toolCallNames) : null,
        },
      })

      // Update conversation timestamp
      await prisma.conversation.update({
        where: { id: activeConversationId },
        data: { updatedAt: new Date() },
      })

      // Fire-and-forget memory extraction (LLM-based via Plan 02)
      extractAndSaveMemory(fullAssistantText, userMessage).catch(err =>
        console.error('Memory extraction failed:', err)
      )

      send('message_complete', { conversationId: activeConversationId })
      controller.close()
    } catch (err) {
      console.error('Chat stream error:', err)
      send('stream_error', { message: 'Something went wrong. Try sending again.' })
      controller.close()
    }
  },
})

return new Response(stream, {
  headers: {
    'Content-Type': 'text/event-stream; charset=utf-8',
    'Cache-Control': 'no-cache, no-transform',
    'Connection': 'keep-alive',
  },
})
```

Wrap the entire POST handler in try-catch. The outer catch returns `Response.json({ error: 'Failed to start chat stream' }, { status: 500 })` for errors before the stream starts.

**BetaToolRunner API adaptation notes:**
- The SDK types show `BetaMessageStream` implements `on('text', ...)` and `on('contentBlock', ...)` — use these directly.
- If `turn.on('text', ...)` doesn't fire text deltas (some SDK versions use different event names), try `turn.on('streamEvent', (event) => { ... })` and check `event.type` for `'content_block_delta'` with `delta.type === 'text_delta'`.
- The `for await (const turn of runner)` loop handles multi-turn tool calls automatically (D-05).
- `max_iterations: 8` is the SDK-native cap — no manual tracking needed.
- Do NOT add `export const runtime = 'edge'` — Prisma requires Node.js runtime (Pitfall 5 from RESEARCH.md).

**Conversation persistence latency note (review concern about sequential DB writes):**
The DB writes (save user message, build context) happen BEFORE the stream starts. This adds ~10-20ms which is imperceptible. The assistant message save happens AFTER the stream completes, so it doesn't affect perceived latency at all.
