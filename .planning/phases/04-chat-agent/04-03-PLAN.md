**components/ChatClient.tsx** — Main stateful shell (`'use client'`):

Props: `{ initialMessages?: Array<{ id: string; role: 'user' | 'assistant'; content: string }>; conversationId?: string | null; pageContext?: string }`

State:
- `messages`: Array of `{ id: string; role: 'user' | 'assistant'; content: string }` — initialized from `initialMessages` or `[]`
- `conversationId`: string | null — initialized from prop
- `streaming`: boolean — true while SSE stream is active
- `streamingText`: string — accumulated text from current stream
- `skeletonVisible`: boolean — true between send and first text_delta
- `toolActivity`: string | null — current tool name being used
- `error`: string | null — stream error message

Refs:
- `scrollRef`: ref to the scroll container div, used to auto-scroll to bottom
- `streamingTextRef`: useRef<string> to avoid stale closure in stream loop (per RESEARCH.md anti-pattern note about React state in while loops)
- `isAtBottomRef`: useRef<boolean> for conditional auto-scroll (Gemini suggestion: use scroll detection to only auto-scroll when user is already at bottom)

**Scroll behavior:**
- Track scroll position with an `onScroll` handler on `scrollRef`: `isAtBottomRef.current = (el.scrollHeight - el.scrollTop - el.clientHeight) < 50`
- Auto-scroll `useEffect`: only scroll to bottom when `isAtBottomRef.current` is true AND (`messages` changes, `streamingText` changes, or `skeletonVisible` changes)
- On send: always force scroll to bottom (user just sent, they want to see the response)

**SSE Stream Reader (addresses review concern #6 — explicit buffer handling):**

`sendMessage` callback:
1. Add user message to `messages` state with a temp `id` (`crypto.randomUUID()` or `Date.now().toString()`)
2. Set `streaming=true`, `skeletonVisible=true`, `error=null`, `toolActivity=null`
3. `fetch('/api/chat', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ conversationId, userMessage: text, pageContext }) })`
4. If response not ok, set error = "Something went wrong. Try sending again." and return
5. Read from `response.body!.pipeThrough(new TextDecoderStream()).getReader()`
6. **Buffer-based SSE parsing (explicit, not naive):**

```typescript
let buffer = ''
while (true) {
  const { value, done } = await reader.read()
  if (done) break

  buffer += value

  // Process complete SSE events only (delimited by \n\n)
  const events = buffer.split('\n\n')
  // Last element may be incomplete — keep it in buffer
  buffer = events.pop() || ''

  for (const eventBlock of events) {
    if (!eventBlock.trim()) continue

    let eventType = 'message'
    let eventData = ''

    for (const line of eventBlock.split('\n')) {
      if (line.startsWith('event: ')) {
        eventType = line.slice(7)
      } else if (line.startsWith('data: ')) {
        eventData = line.slice(6)
      }
    }

    if (!eventData) continue

    try {
      const payload = JSON.parse(eventData)

      switch (eventType) {
        case 'text_delta':
          setSkeletonVisible(false)
          streamingTextRef.current += payload.delta
          setStreamingText(streamingTextRef.current)
          break
        case 'tool_activity':
          setToolActivity(payload.toolName)
          break
        case 'message_complete':
          setMessages(prev => [...prev, {
            id: payload.conversationId || Date.now().toString(),
            role: 'assistant' as const,
            content: streamingTextRef.current,
          }])
          setStreamingText('')
          streamingTextRef.current = ''
          setToolActivity(null)
          setStreaming(false)
          setSkeletonVisible(false)
          if (payload.conversationId) {
            setConversationId(payload.conversationId)
          }
          break
        case 'stream_error':
          setError(payload.message || 'Something went wrong. Try sending again.')
          setStreaming(false)
          setSkeletonVisible(false)
          setToolActivity(null)
          break
      }
    } catch {
      // Malformed JSON — skip this event
    }
  }
}
```

7. On fetch error (network), set error message and reset streaming states.

**app/chat/page.tsx** — Server component:
- Import ChatClient
- Read `searchParams` for `context` query param (for context-aware FAB navigation per D-01)
- Optionally load the most recent conversation from Prisma (if no context param, resume last conversation; if context param present, start fresh per RESEARCH.md Open Question 3)
- Pass `initialMessages`, `conversationId`, and `pageContext` to ChatClient

```typescript
import { prisma } from '@/lib/db'
import ChatClient from '@/components/ChatClient'

export default async function ChatPage({ searchParams }: { searchParams: Promise<{ context?: string }> }) {
  const { context } = await searchParams

  let initialMessages: Array<{ id: string; role: string; content: string }> = []
  let conversationId: string | null = null

  // If no context param, resume most recent conversation
  if (!context) {
    const lastConversation = await prisma.conversation.findFirst({
      orderBy: { updatedAt: 'desc' },
      include: {
        messages: {
          orderBy: { createdAt: 'asc' },
          take: 50,
        },
      },
    })

    if (lastConversation) {
      conversationId = lastConversation.id
      initialMessages = lastConversation.messages.map(m => ({
        id: m.id,
        role: m.role,
        content: m.content,
      }))
    }
  }

  // Parse context param into human-readable string
  let pageContext: string | undefined
  if (context) {
    // Format: "trip:clxyz123" or "gear:cla456" or "spot:clb789"
    const [type, id] = context.split(':')
    if (type === 'trip' && id) {
      const trip = await prisma.trip.findUnique({ where: { id }, select: { name: true } })
      pageContext = trip ? `User is viewing Trip: ${trip.name}` : undefined
    } else if (type === 'gear' && id) {
      const gear = await prisma.gearItem.findUnique({ where: { id }, select: { name: true } })
      pageContext = gear ? `User is viewing Gear: ${gear.name}` : undefined
    } else if (type === 'spot' && id) {
      const loc = await prisma.location.findUnique({ where: { id }, select: { name: true } })
      pageContext = loc ? `User is viewing Spot: ${loc.name}` : undefined
    }
  }

  return <ChatClient initialMessages={initialMessages} conversationId={conversationId} pageContext={pageContext} />
}
```
