## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafaf9` light / `#0c0a09` dark (stone-50/stone-950) | Page background, chat scroll area |

| Accent (10%) | `#d97706` light / `#f59e0b` dark (amber-600/amber-500) | Send button, active nav tab (chat icon), focus ring on input, skeleton shimmer highlight |
| Destructive | `#dc2626` light / `#ef4444` dark (red-600/red-500) | Confirmation prompt for agent delete actions only |

**Source:** `app/globals.css` CSS custom properties вҖ” `--color-bg-primary`, `--color-bg-surface`, `--color-accent`, `--color-danger`.

Accent reserved for: send button background, active BottomNav chat tab icon, input focus ring, tool-activity badge accent variant. Never used on agent message bubbles, timestamps, or scroll indicators.

**Chat bubble color split:**
- User messages: `bg-amber-600 text-white` (light) / `bg-amber-500 text-stone-900` (dark) вҖ” right-aligned
- Agent messages: `bg-white border border-stone-200` (light) / `bg-stone-900 border border-stone-700` (dark) вҖ” left-aligned
- Tool activity indicator: `bg-stone-100 text-stone-500` (light) / `bg-stone-800 text-stone-400` (dark) вҖ” left-aligned, italic label

## Focal Point

**Primary visual anchor (`/chat` screen):** The amber send button in the input bar is the only high-chroma element on screen. All other surfaces are stone neutrals. The eye is drawn to the input area вҖ” this reinforces that the action is typing and sending. No competing accent elements appear above the input bar.

## Interaction States

### Message Loading (Skeleton Bubble)

- Display: left-aligned gray placeholder bubble, same shape as agent bubble
- Animation: `.skeleton` shimmer class from `globals.css` (already defined)
- Timing: shown immediately on send, before first streaming token arrives
- Transition: skeleton fades out, real bubble fades in as text streams character by character
- **Source:** CONTEXT.md D-02 вҖ” "skeleton bubble loading state"

### Streaming Response

- Text appears token by token using SSE
- Bubble height expands naturally as text grows (no fixed height)
- No typing indicator dots вҖ” skeleton bubble covers the pre-stream state
- Cursor: blinking `|` appended to streaming text until stream completes

### Tool Activity Indicator

- Shown as a small left-aligned status chip below the skeleton bubble while tools run
- Format: `ToolActivityIndicator` вҖ” `badge muted` variant + italic label text
- Examples: "Checking your gear...", "Looking up weather...", "Searching knowledge base..."
- Disappears when agent response starts streaming
- **Source:** CONTEXT.md В§Specifics вҖ” "agent should show what it's doing"

### Confirm Before Destructive Agent Actions

- When agent plans to delete: render an inline confirmation card inside the chat, not a modal
- Card contains: description of what will be deleted + two buttons: "Keep [item]" (secondary) and "Delete [item]" (danger)
- Agent waits for user response before executing the delete
- **Source:** CONTEXT.md D-04

### Input Area

- Textarea (not single-line input) вҖ” auto-expands up to 5 lines, then scrolls
- `Enter` sends message; `Shift+Enter` inserts newline
- Send button: disabled + `opacity-50` when textarea is empty or stream is in progress
- On send: clear textarea, scroll to bottom, show skeleton bubble immediately

### Empty State (No Messages)

- Centered vertically in scroll area
- Uses `EmptyState` component with emoji `рҹҸ•пёҸ`, title and description per Copywriting Contract below
- No action button in empty state вҖ” chat begins by typing

## Accessibility Contract

- Send button: `aria-label="Send message"` when icon-only
- SkeletonBubble: `aria-label="Loading response"` + `role="status"`
- Tool activity: `aria-live="polite"` so screen readers announce tool changes
- Chat scroll area: `aria-label="Conversation"` + `role="log"` with `aria-live="polite"` and `aria-relevant="additions"`
- Input textarea: `aria-label="Message"` + `placeholder="Message..."` вҖ” never placeholder as label
- Destructive confirm buttons: destructive button receives focus on render
