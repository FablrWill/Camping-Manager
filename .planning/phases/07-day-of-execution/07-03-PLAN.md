Steps:
- Read `{ tripId }` from request body
- Fetch trip: `prisma.trip.findUnique({ where: { id: tripId }, include: { packingItems: { include: { gear: true } }, mealPlan: true, vehicle: true, location: true, departureChecklist: true } })`
- If trip not found, return 404
- Resolve emergency contact (per D-08): `trip.emergencyContactEmail ?? settings?.emergencyContactEmail` where settings = `prisma.settings.findUnique({ where: { id: 'user_settings' } })`
- Resolve emergency contact name: `trip.emergencyContactName ?? settings?.emergencyContactName`
- If no emergency contact email, return 400 `{ error: 'No emergency contact configured. Add one in Settings.' }`
- If no GMAIL_USER or GMAIL_APP_PASSWORD env vars, return 400 `{ error: 'Gmail credentials not configured. Set GMAIL_USER and GMAIL_APP_PASSWORD in .env' }`
- Build packed gear summary: group packed items by category, format as "Shelter: tent, tarp, stakes; Cooking: stove, pot, utensils; ..."
- Build Google Maps link if location has coordinates: `https://www.google.com/maps?q=${lat},${lon}` per D-10
- Build checklist status string (addresses review concern): if trip.departureChecklist exists, parse its result JSON, count total items and checked items, format as "X of Y departure tasks completed". If no checklist, use "Departure checklist not yet generated".
- Call `composeFloatPlanEmail` with trip data including checklistStatus
- Append Google Maps link to email body if coordinates exist: add a line "\nDestination: {location name} - {google maps link}"
- CRITICAL: Call `sendFloatPlan` with `text` parameter (NOT `html`): `sendFloatPlan({ to: recipientEmail, toName: recipientName, subject: emailResult.subject, text: emailBody })`
- Log to FloatPlanLog (fire-and-forget after successful send -- don't let log failure block response):
  ```typescript
  prisma.floatPlanLog.create({
    data: { tripId, sentTo: recipientEmail, sentToName: recipientName, subject: emailResult.subject, body: emailBody }
  }).catch(err => console.error('Failed to log float plan:', err));
  ```
- Return `{ success: true, sentTo: recipientEmail, sentToName: recipientName, sentAt: new Date().toISOString() }`
- Error handling: try-catch, 422 for Claude parse errors, 500 for send errors with user-friendly message "Could not send email - check Gmail config in Settings"
