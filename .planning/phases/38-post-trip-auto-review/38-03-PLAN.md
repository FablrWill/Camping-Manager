    Add `reviewedAt: string | null` to the TripData interface in both files. TypeScript will catch any usages that need updating.

    - Add `onReview: (tripId: string) => void` to TripCardProps interface
    - For past trips where `trip.reviewedAt === null`: render a "Review trip" button below the card metadata. Style with amber/orange accent (e.g. `bg-amber-50 border border-amber-200 text-amber-800` or similar) with a clipboard or checkmark icon. Clicking calls `onReview(trip.id)`.

    - Add `reviewingTripId` state: `const [reviewingTripId, setReviewingTripId] = useState<string | null>(null)`
    - Pass `onReview={(id) => setReviewingTripId(id)}` to each TripCard

    - Render `<TripReviewModal>` when `reviewingTripId !== null` — find the trip from the trips array and build the props:

      - `meals`: if trip.mealPlan exists, map meals to `{ mealId: m.id, mealPlanId: trip.mealPlan.id, name: m.name, dayLabel: \`Day ${m.day} - ${m.slot}\` }`, else `[]`
      - `locationId`: trip.locationId
      - `locationName`: trip.location?.name ?? null
      - `existingNotes`: trip.notes
    - `onComplete`: receives reviewedAt string → update trips state immutably: `setTrips(prev => prev.map(t => t.id === reviewingTripId ? { ...t, reviewedAt } : t))`; then `setReviewingTripId(null)`
    - `onClose`: `setReviewingTripId(null)`
