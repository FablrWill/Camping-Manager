# Phase 44 Plan 00: Google Maps List Import

## Objective

Extend the existing Google Maps import feature to support **Google Takeout "Saved Places.json"** file upload and improve the text parser to handle Google Maps shared list format. Add a file input alongside the existing text paste UI, and add parser unit tests.

## Context

The backend API (`/api/import/google-maps`) and frontend UI in `spots-client.tsx` already exist for text/URL pasting. The gap is:
- No support for Google Takeout structured JSON format (`Saved Places.json`)
- No file upload option in the UI (only textarea paste)
- No unit tests for the parser

Google Takeout "Saved Places.json" format:
```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "geometry": { "coordinates": [-82.5515, 35.5951], "type": "Point" },
      "properties": {
        "Google Maps URL": "https://maps.google.com/?cid=...",
        "Location": {
          "Address": "123 Forest Rd, Asheville, NC",
          "Business Name": "Pisgah National Forest",
          "Country Code": "US",
          "Geo Coordinates": { "Latitude": "35.5951", "Longitude": "-82.5515" }
        },
        "Published": "2024-01-15T10:30:00Z",
        "Title": "Pisgah Campsite",
        "Updated": "2024-01-15T10:30:00Z"
      },
      "type": "Feature"
    }
  ]
}
```

## Tasks

### Task 1: Add Saved Places JSON parser to the API route

`type="auto"`

Extend `app/api/import/google-maps/route.ts` to:
- Accept multipart form data (for file upload) in addition to JSON body
- Export `parseLocationsFromText` as a named export (for testing)
- Add `parseSavedPlacesJson(json: unknown): ParsedLocation[]` function that handles the GeoJSON FeatureCollection format from Google Takeout
- Add a POST endpoint that handles both `Content-Type: application/json` (existing) and `Content-Type: multipart/form-data` (new, for file upload)
- When a `.json` file is uploaded, detect if it's the Saved Places format and parse it
- Preserve existing text/URL paste behavior

**Done criteria:**
- `parseSavedPlacesJson` correctly extracts name, lat, lon, and address from the Takeout format
- Coordinates from `geometry.coordinates` and `Location.Geo Coordinates` both handled
- Falls back gracefully if format doesn't match
- Existing text paste behavior unchanged

### Task 2: Add file upload input to the Spots import UI

`type="auto"`

Update `app/spots/spots-client.tsx` to:
- Add a file input (`<input type="file" accept=".json">`) above the textarea in the Google Maps import panel
- When a JSON file is selected, POST it as multipart form data to `/api/import/google-maps`
- Show import progress and results (same pattern as GPX import)
- Update the description text to mention "Or upload a Saved Places.json file from Google Takeout"
- Keep the textarea paste option working

**Done criteria:**
- User can select a `.json` file from their device
- File is posted to the API
- Success/error messages shown
- Textarea paste still works

### Task 3: Add unit tests for the Google Maps parser

`type="auto"`

Create `lib/__tests__/google-maps-import.test.ts` with tests for:
- `parseLocationsFromText`: URL patterns, decimal degree pairs, name+coord pattern, deduplication
- `parseSavedPlacesJson`: valid Takeout format, coordinates from geometry, coordinates from Location, title/address extraction, invalid format fallback

**Done criteria:**
- All tests pass with `npm run test -- lib/__tests__/google-maps-import.test.ts`
- Tests cover happy path and edge cases
