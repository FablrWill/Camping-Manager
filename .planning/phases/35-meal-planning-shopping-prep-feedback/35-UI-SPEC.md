## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge padding, slot color dot |
| sm | 8px | Compact row padding, gap between feedback buttons |
| md | 16px | Card internal padding (`p-4`), tab content padding |
| lg | 24px | Section gaps between tab panels |
| xl | 32px | Page-level vertical rhythm |
| 2xl | 48px | Major section breaks (not expected in this phase) |
| 3xl | 64px | Page-level spacing (not expected in this phase) |

Exceptions:
- Touch targets for 👍/👎 buttons: minimum 44px height (phone-friendliness, D-02)
- Checkbox tap area in shopping list: minimum 44px height per row

Source: Existing `MealPlanClient.tsx` uses `p-4` (16px) as base card padding. Spacing scale confirmed from `globals.css` design tokens.

**Accent reserved for:**
- "Generate Shopping List" primary button (CTA)
- "Generate Prep Guide" primary button (CTA)

## Interaction Contracts

### Tab Bar — Plan / Shopping / Prep

- Renders inside `MealPlanClient` at the top of the card, below the header row.
- 3 tabs: "Plan" | "Shopping" | "Prep"
- Active tab: `border-b-2 border-amber-500 text-stone-900 dark:text-stone-100 font-semibold`
- Inactive tab: `text-stone-500 dark:text-stone-400` with `hover:text-stone-700`
- Tab bar background: `bg-stone-50 dark:bg-stone-800/50 border-b border-stone-100 dark:border-stone-800`
- Tab switching is instant client-side state — no loading state between tabs.
- Default tab on mount: "Plan" (preserves existing behavior).

### MealFeedbackButton (D-01, D-02, D-03)

- Placement: bottom of each meal card, left-aligned, below the ingredients/instructions block.
- Default state: two icon buttons side by side — `ThumbsUp` and `ThumbsDown` from Lucide, size 16.
- Button size: `min-h-[44px] min-w-[44px]` to meet touch target requirement. Use `px-3 py-2` with the icon centered.
- Unrated state: both buttons use `ghost` variant styling (`text-stone-400 hover:text-stone-600`).
- After tapping 👍: button fills `text-green-600 dark:text-green-400 bg-green-50 dark:bg-green-900/30 rounded-md`. 👎 returns to ghost.
- After tapping 👎: button fills `text-red-500 dark:text-red-400 bg-red-50 dark:bg-red-900/30 rounded-md`. 👍 returns to ghost.
- Tapping either thumb reveals a `<textarea>` directly below the buttons with `placeholder="Add a note (optional)"`.
- Textarea: `w-full text-sm rounded-md border border-stone-200 dark:border-stone-700 bg-white dark:bg-stone-800 p-2 mt-2 resize-none` with 2 rows.
- Save is automatic on blur (PATCH to API). No "Save note" button needed — saves silently.
- If a rating already exists for this meal, pre-fill the active state on mount.

### Shopping List — ShoppingListClient (D-04 through D-07)

**Empty state (no shopping list generated yet):**
- Render inside Shopping tab panel.
- Single centered block: `ShoppingCart` icon (24px, `text-stone-300`), heading "No shopping list yet", body "Tap Generate to build your list from the meal plan.", then "Generate Shopping List" primary button.

**Generating state:**
- Replace button with spinner + "Claude is building your shopping list..." text.
- 3 skeleton rows: `animate-pulse bg-stone-100 dark:bg-stone-800 rounded h-8 w-full`

**List rendered state:**
- Header row: "Shopping List" heading (text-sm semibold) on the left; "Copy List" ghost button with `Clipboard` icon on the right. "Regenerate" ghost button with `RotateCcw` icon next to it.
- Items grouped by category (e.g., "Produce", "Pantry", "Dairy"). Category label: `text-xs font-medium text-stone-500 uppercase tracking-wider mt-4 mb-2`.
- Each item row: `flex items-center gap-3 py-2 border-b border-stone-50 dark:border-stone-800`.
  - Checkbox: `w-5 h-5 rounded border-stone-300 accent-amber-500` (native checkbox styled with Tailwind).
  - Item name: `text-sm text-stone-800 dark:text-stone-200`. When checked: `line-through text-stone-400`.
  - Quantity + unit: `text-xs text-stone-400 ml-auto shrink-0`.
- Checked state persisted to API on every toggle (PATCH per item). Optimistic update.
- Copy button behavior: copies plain-text list to clipboard, then button label flashes "Copied!" in `text-amber-600` for 1500ms, returns to "Copy List".

**Plain-text clipboard format (Claude's Discretion — resolved here):**
```
Shopping List — [Trip Name]

PRODUCE
- 2 lbs sweet potatoes
- 1 bunch cilantro

PANTRY
- 1 can black beans
- olive oil (have some)

[...etc]
```

### Prep Guide — PrepGuideClient (D-08, D-09)

**Empty state (no prep guide generated yet):**
- `ClipboardList` icon (24px, `text-stone-300`), heading "No prep guide yet", body "Generate a step-by-step guide for what to do before you leave and what to cook at camp.", then "Generate Prep Guide" primary button.

**Generating state:**
- Spinner + "Claude is writing your prep guide..."
- 4 skeleton rows.

**Guide rendered state:**
- Two sections: "Before You Leave" and "At Camp".
- Section heading: `text-sm font-semibold text-stone-900 dark:text-stone-100 flex items-center gap-2`. Use `Home` icon for "Before You Leave", `Flame` icon for "At Camp".
- Each step: `flex gap-3 py-2 text-sm text-stone-700 dark:text-stone-300`. Step number in `text-xs text-stone-400 font-mono w-5 shrink-0`.
- "At Camp" steps include the day + meal slot as a label: `text-xs text-amber-600 dark:text-amber-400 font-medium` above the step text.
- "Regenerate Guide" ghost button with `RotateCcw` icon in the top-right of the section header.

### Dashboard Meal Plan Card (D-10, D-11)

- Positioned below the existing "Next Trip" card in DashboardClient (or merged into it if the same trip — see below).
- Shows soonest upcoming trip with a future start date.
- If same trip as existing trip card: add a second line inside the existing amber card: `text-xs text-amber-700 dark:text-amber-300 mt-1` showing meal plan status.
- If no upcoming trip: card is not rendered.

**Status line copy logic:**
- No meal plan → "No meal plan yet — tap to generate"
- Meal plan generated, no shopping list → "Meal plan ready — shopping list pending"
- Shopping list generated, items remain unchecked → "Shopping list ready — [N] items to get"
- All shopping items checked → "All stocked up — ready to roll"
- Card tapping navigates to `/trips/[id]/prep` (meals tab).
