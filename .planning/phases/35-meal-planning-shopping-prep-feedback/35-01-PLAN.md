**1. app/api/trips/[id]/meal-plan/shopping-list/route.ts**

GET handler:
- Extract tripId from params
- Query `prisma.mealPlan.findUnique({ where: { tripId }, select: { id: true } })`
- If no meal plan, return `NextResponse.json({ items: [] })`
- Query `prisma.shoppingListItem.findMany({ where: { mealPlanId }, orderBy: [{ category: 'asc' }, { item: 'asc' }] })`
- Return `NextResponse.json({ items })`

POST handler (generate/regenerate per D-04):
- Extract tripId from params
- Query meal plan with all meals: `prisma.mealPlan.findUnique({ where: { tripId }, include: { meals: true } })`
- If no meal plan, return 404: `NextResponse.json({ error: 'No meal plan found for this trip' }, { status: 404 })`
- Parse each meal's `ingredients` JSON string into `Array<{ item, quantity, unit }>`
- Call `generateShoppingList({ tripName: trip name from meal plan, meals: parsed meal data })`
- Implement D-07 merge logic:
  a. Load existing checked items: `prisma.shoppingListItem.findMany({ where: { mealPlanId: mealPlan.id, checked: true }, select: { item: true } })`
  b. Build Set of lowercase checked names: `new Set(existing.map(e => e.item.toLowerCase()))`
  c. Delete all old items: `prisma.shoppingListItem.deleteMany({ where: { mealPlanId: mealPlan.id } })`
  d. Create new items: `prisma.shoppingListItem.createMany({ data: result.items.map(ni => ({ mealPlanId: mealPlan.id, item: ni.item, quantity: ni.quantity, unit: ni.unit, category: ni.category, checked: checkedNames.has(ni.item.toLowerCase()) })) })`
  e. Steps c+d should be in a `prisma.$transaction([...])`
- Fetch and return the newly created items
- All items start unchecked per D-05 unless they match a previously checked name per D-07
