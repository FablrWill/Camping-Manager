<objective>
Build the agent jobs infrastructure: Prisma model, three API routes (list/create, detail, results), dashboard badge for unread results, and gear enrichment trigger on save.

Purpose: AGENT-01 — the job queue is the foundation for all Mac mini background processing. Future job types (deal checks, weather analysis) slot into this infrastructure.
Output: AgentJob table, REST API for job lifecycle, AgentJobsBadge component, gear enrichment trigger.
</objective>

## Tasks

### 1. Add AgentJob model to Prisma schema
- Add model with: id (cuid), type (String), status (String, default "pending"), triggeredBy (String, default "manual"), payload (String — JSON), result (String? — JSON), readAt (DateTime?), createdAt (DateTime), completedAt (DateTime?)
- Add indexes on status and createdAt for efficient polling
- Run `npx prisma migrate dev --name add-agent-job`

### 2. Create job list + create API route
- `app/api/agent/jobs/route.ts`
- GET: query by `?status=` filter, returns jobs ordered by createdAt desc
- POST: accepts `{ type, payload, triggeredBy }`, validates required fields, creates job, returns 201

### 3. Create job detail API route
- `app/api/agent/jobs/[id]/route.ts`
- GET: fetch single job by id, return 404 if not found

### 4. Create results API route
- `app/api/agent/results/route.ts`
- POST: accepts `{ jobId, result }`, finds the job, sets status="done", result=JSON.stringify(result), completedAt=now()
- For gear_enrichment type: also update the GearItem with enriched fields (brand, notes, weight)
- Return 404 if job not found, 400 if already done

### 5. Build AgentJobsBadge component
- Client component showing count of jobs where status="done" AND readAt IS NULL
- Polls on mount via GET /api/agent/jobs?status=done&unread=true
- Click expands a dropdown showing recent results
- "Mark all read" button PATCHes readAt on those jobs
- Add PATCH to jobs/[id] route for marking read

### 6. Integrate badge into Dashboard
- Import AgentJobsBadge into DashboardClient
- Place in header area near the title
- Pass no props — badge fetches its own data

### 7. Trigger gear enrichment on gear save
- In GearClient, after a successful POST (new gear), check if brand and notes are both empty
- If so, fire a background POST to /api/agent/jobs with type="gear_enrichment" and payload containing gearItemId, name, category
- Fire-and-forget — don't block the UI

### 8. Wire dashboard server component
- In app/page.tsx, fetch pending + unread-done job counts and pass to DashboardClient as props (for initial render, badge can hydrate from props)

### 9. Build verification
- Run `npm run build` and fix any type errors
