   - Read the PDF file as a Buffer: `readFileSync(filePath)`.
   - Extract text using `pdfParse(buffer)` — returns `{ text, numpages, info }`.
   - Use `info.Title` or `basename(filePath, '.pdf')` as the document title.
   - Split extracted text into chunks using paragraph splitting (split on `\n\n`, accumulate to 256-512 token target — reuse the same `estimateTokens` logic from `ingest.ts`).
   - Import `estimateTokens` from `../ingest` (or duplicate the simple function — `encode(text).length`).
   - Set metadata:
     ```typescript

       confidence: 'official',
       verifyFlag: false,
     };
     ```
   - Return array of RawChunk with sequential chunkIdx, source = filePath.
   - Export `chunkPdf`.

3. Create `lib/rag/parsers/web.ts` — web page scraping and chunking:
   ```typescript
   import * as cheerio from 'cheerio';
   import { RawChunk, ChunkMetadata } from '../types';
   ```

   **Function `chunkWebPage(url: string): Promise<RawChunk[]>`:**
   - Fetch the URL using `fetch(url)` (Node 18+ built-in fetch).
   - If response is not ok, throw: `"Failed to fetch ${url}: ${response.status}"`.
   - Parse HTML with cheerio: `const $ = cheerio.load(html)`.
   - Extract page title: `$('title').text()` or `$('h1').first().text()`.
   - Remove non-content elements: `$('script, style, nav, footer, header, .sidebar, #menu').remove()`.
   - Extract main content: prefer `$('main').text()` or `$('article').text()` or `$('#content').text()`, fallback to `$('body').text()`.
   - Clean extracted text: collapse whitespace, trim.
   - Split into chunks at paragraph boundaries (double newlines or sentence boundaries), targeting 256-512 tokens.
   - Set metadata:
     ```typescript
     const metadata: ChunkMetadata = {
       topic: pageTitle,
       category: 'external',
       confidence: 'official',
       verifyFlag: true, // web content may change — flag for freshness checks
     };
     ```
   - Return array of RawChunk with source = url, sequential chunkIdx.
   - Export `chunkWebPage`.

4. Update `lib/rag/ingest.ts` — modify `ingestFile` to route by file type:
   - Import `chunkPdf` from `./parsers/pdf`.
   - Import `chunkWebPage` from `./parsers/web`.
   - Update `ingestFile` function:
     ```typescript
     export async function ingestFile(filePath: string): Promise<RawChunk[]> {
       // URL detection
       if (filePath.startsWith('http://') || filePath.startsWith('https://')) {
         return chunkWebPage(filePath);
       }

       const ext = filePath.split('.').pop()?.toLowerCase();
       switch (ext) {
         case 'md':
           const content = readFileSync(filePath, 'utf-8');
           return chunkMarkdown(content, filePath);
         case 'pdf':
           return chunkPdf(filePath);
         default:
           throw new Error(`Unsupported file type: .${ext}. Supported: .md, .pdf, or http/https URL`);
       }
     }
     ```
   - Note: `ingestFile` is now async (returns Promise) because PDF parsing and web fetching are async. Update the signature and any callers.
   - Also export `estimateTokens` so the parsers can import it (or extract it to a shared util — `lib/rag/tokenize.ts` with `export function estimateTokens(text: string): number`).
