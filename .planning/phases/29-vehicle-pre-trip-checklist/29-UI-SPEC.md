## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `py-2`) |
| md | 16px | Default element spacing (`px-4`, `py-4`) |
| lg | 24px | Section padding (`py-6`, `space-y-6`) |

- Section header button: `px-4 py-3` (48px effective height on mobile). Source: `TripPrepSection.tsx` line 39.

**Accent reserved for:**
- Progress bar fill (amber-600 dark:amber-500)
- "Generate Vehicle Checklist" primary CTA button (`variant="primary"` in Button.tsx = amber)
- Checked item count label in progress summary ("X of Y items done")
- Skeleton loading gradient highlight
- Focus rings on interactive inputs (`--color-border-focus: amber-400`)

Accent is NOT used on: section titles, body text, checklist item text, card backgrounds, or status badges (status badges use emerald/amber/stone per PrepStatus type).

**Focal points by state:**
- Empty state focal point: amber "Generate Vehicle Checklist" button — the sole interactive element, amber color draws the eye immediately.
- Loaded state focal point: amber progress bar + bold item count label ("X of Y items done") in amber-600 — positioned above the item list, visually dominant against the stone card background.

## Interaction Contracts

### Checklist Item Row
- Renders as a tappable row with a checkbox on the left and item text on the right
- Minimum height: 44px (mobile touch target)
- Checkbox: native `<input type="checkbox">` styled with accent color when checked — amber-600 fill, stone-200 border when unchecked
- Item text: `text-sm` (14px) weight 400. When `checked=true`: apply `line-through text-stone-400 dark:text-stone-500`
- Row background: white / stone-900, divided by `divide-y divide-stone-100 dark:divide-stone-800`
- Tap/click fires optimistic state update immediately, then sends PATCH in background (fire-and-forget pattern from `DepartureChecklistClient.tsx` line 222)
- Rows are contained in a card: `bg-white dark:bg-stone-900 rounded-xl border border-stone-200 dark:border-stone-700 divide-y ... overflow-hidden`

### Progress Bar
- Shows when checklist result is loaded and `items.length > 0`
- Amber fill bar on stone-200/stone-700 track: `bg-amber-600 dark:bg-amber-500 rounded-full h-2`
- Label above bar: `"{checkedCount} of {totalCount} items done"` — 14px text-sm font-semibold (weight 600) amber-600 dark:amber-400
- Percentage on the right: `text-sm text-stone-400 dark:text-stone-500`
- Bar animates width on check-off: `transition-all duration-300`

### Loading Skeleton
- Shows while `vehicleChecklistLoading=true`
- Introductory text: "Claude is building your checklist..." — 14px stone-500
- 8–12 skeleton rows (matches expected item count): `animate-pulse bg-stone-100 dark:bg-stone-800 rounded h-10 w-full`
- No section headers in skeleton — flat list per D-02 (no grouping in this phase)

### Generate Button (empty state)
- Rendered via `EmptyState` component's `action` prop
- Label: "Generate Vehicle Checklist"
- Disabled when `vehicleChecklistLoading=true`
- Color: amber-600 link style (matches EmptyState action button pattern in `EmptyState.tsx` line 24)

### Regenerate Button (loaded state)
- `<Button variant="ghost" size="sm">Regenerate Checklist</Button>`
- Positioned below the item list, left-aligned
- Click opens `ConfirmDialog` — does not regenerate directly
- Disabled when offline (matches departure checklist pattern)

### Section Summary Text (collapsed state)
- Passed as `summary` prop to `TripPrepSection`
- Format: `"{checkedCount} of {totalCount} done"` when checklist exists
- Format: `"Not generated"` when no checklist stored
- Format: `"No vehicle assigned"` when trip has no vehicle

### Status Logic
- `not_started` — `vehicleChecklistResult` is null on the Trip
- `in_progress` — result exists AND at least one item is unchecked
- `ready` — result exists AND all items are checked
