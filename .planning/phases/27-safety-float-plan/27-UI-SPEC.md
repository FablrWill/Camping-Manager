### Email Body Template (plain-text, deterministic)

```
[Name] is camping at [destination] from [start] to [end].
Vehicle: [vehicle name].
Expected return: [end date].
Notes: [notes]       ← omit entire line if no notes
Map: [Google Maps link]   ← omit entire line if no coordinates
```

Missing field handling (Claude's discretion per CONTEXT.md):
- No vehicle: omit the "Vehicle:" line entirely
- No notes: omit the "Notes:" line entirely
- No coordinates: omit the "Map:" line entirely
- No destination name: use trip name as fallback

Source: CONTEXT.md D-07, D-08

## Interaction States

The float plan button in `DepartureChecklistClient.tsx` has 5 states. All already implemented. The API refactor must preserve the same response shape:

| State | Condition | Button Appearance |
|-------|-----------|-------------------|
| Ready | contact set, online | Amber primary, "Send Float Plan" |
| Sending | `sendingFloatPlan` true | Disabled, spinner (loading prop), "Sending..." |
| Offline | `isOnline` false | Disabled, "Connect to send" |
| No contact | `!resolvedContactEmail` | Disabled, amber warning strip below |
| Sent | `floatPlanSent` truthy | Replaced by green CheckCircle + confirmation text |

API response shape the route must return on success (unchanged from existing):
```json
{ "sentTo": "email@example.com", "sentToName": "Name", "sentAt": "ISO timestamp" }
```
