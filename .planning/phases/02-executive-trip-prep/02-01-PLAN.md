The handler receives `params: { id: string }`. Query the trip:
```typescript
const trip = await prisma.trip.findUnique({
  where: { id: params.id },
  include: {
    location: { select: { id: true, name: true, latitude: true, longitude: true } },
    vehicle: { select: { id: true, name: true } },
    packingItems: { select: { packed: true } },
  },

```

If no trip, return `NextResponse.json({ error: 'Trip not found' }, { status: 404 })`.

Compute section status for each entry in `PREP_SECTIONS`:

**Weather section:**
- If no `trip.location` or no `trip.location.latitude` → status: `not_started`, summary: `No location set`
- Otherwise, try `fetchWeather(trip.location.latitude, trip.location.longitude, trip.startDate, trip.endDate)`. Wrap in try/catch.
- If weather data returned: check for alerts with severity `warning`. If alerts → `in_progress`, summary includes alert count. If no alerts → `ready`, summary shows high/low range. If fetch fails → `not_started`, summary: `Weather unavailable`.
- `data` field: `{ days, alerts, elevation, locationName: trip.location.name }` (or null if failed)

**Packing section:**
- Query `trip.packingItems` (already included).
- `const total = trip.packingItems.length`
- `const packed = trip.packingItems.filter(i => i.packed).length`
- If `total === 0` → `not_started`, summary: `No packing list generated`
- If `packed === total` → `ready`, summary: `${packed}/${total} items packed`
- Else → `in_progress`, summary: `${packed}/${total} items packed`

**Meals section:**
- Check `trip.mealPlanGeneratedAt`:
- If not null → `ready`, summary: `Meal plan generated` with relative date
- If null → `not_started`, summary: `No meal plan yet`

**Power section:**
- Query `prisma.gearItem.count({ where: { isWishlist: false, wattage: { not: null } } })`
- If count > 0 → `in_progress`, summary: `${count} powered devices tracked`
- If 0 → `not_started`, summary: `No powered gear in inventory`
- (Power budget is calculated on-demand, so we use a simpler heuristic for the badge)

Build `PrepState` with `overallReady = sections.every(s => s.status === 'ready')`.

Return `NextResponse.json(prepState)`.

Use try/catch with `console.error('Failed to fetch prep state:', error)` and 500 response on failure.

**3. Update `components/PackingList.tsx` — persist checkbox state:**

Add an async function `togglePacked` that calls PATCH `/api/packing-list/items`:
```typescript
async function togglePacked(categoryAndItem: string, gearId: string | undefined, newChecked: boolean) {
  setChecked(prev => ({ ...prev, [categoryAndItem]: newChecked }))
  if (gearId) {
    try {
      await fetch('/api/packing-list/items', {
        method: 'PATCH',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ tripId, gearId, packed: newChecked }),
      })
    } catch (err) {
      console.error('Failed to save packed state:', err)
      // Revert optimistic update
      setChecked(prev => ({ ...prev, [categoryAndItem]: !newChecked }))
    }
  }
}
```

Replace the existing `onChange` handler on the checkbox input that does `setChecked(prev => ({ ...prev, [key]: !prev[key] }))` with a call to `togglePacked(key, item.gearId, !checked[key])`.

Note: The packing list items from Claude include a `gearId` field when they map to inventory. Items manually added won't have a `gearId` — those stay local-only, which is fine.
