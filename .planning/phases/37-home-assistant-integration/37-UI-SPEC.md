## Color

Project palette is defined in `app/globals.css` as CSS custom properties. Use these tokens throughout; do not hardcode hex values in components.

| Role | Light Value | Dark Value | CSS Token | Usage |
|------|------------|------------|-----------|-------|
| Dominant (60%) | `#fafaf9` (stone-50) | `#0c0a09` (stone-950) | `--color-bg-primary` | Page background |
| Secondary (30%) | `#ffffff` / `#f5f5f4` | `#1c1917` / `#292524` | `--color-bg-surface` / `--color-bg-surface-alt` | Cards, settings section, entity picker rows |
| Accent (10%) | `#d97706` (amber-600) | `#f59e0b` (amber-500) | `--color-accent` | Reserved list below |
| Destructive | `#dc2626` (red-600) | `#ef4444` (red-500) | `--color-danger` | Disconnect / remove HA config actions only |

Accent (`--color-accent`) reserved for:
- "Test Connection" button (primary action, the one actionable CTA per settings section)
- "Save" button in HA settings section when HA fields are dirty
- "Full View" link/button on the dashboard Campsite card (links to /campsite)
- Live refresh indicator icon on /campsite when actively polling

Accent is NOT used for: entity rows, domain group labels, stale/offline state indicators, or entity picker checkboxes (those use stone border color).

### /campsite Dark Mode Default

The /campsite route uses dark mode by default (D-08b: "dark mode by default — easier outdoor reading"). Apply the `.dark` class on the /campsite page root element unconditionally — do not tie it to the user's global dark mode toggle. Light mode is not supported on /campsite.

Dark surface colors for /campsite sensor cards: `--color-bg-surface` (`#1c1917` stone-900). Card border: `--color-border` (`#44403c` stone-700).

### Status Colors

| State | Color | Token |
|-------|-------|-------|
| Connected / success | emerald-500 `#10b981` | `--color-success` |
| Offline / error | red-500 `#ef4444` | `--color-danger` |
| Stale data (cached) | amber-500 `#f59e0b` | `--color-warning` |
| Unavailable / unknown entity state | stone-400 `#a8a29e` | `--color-text-tertiary` |

## Component Inventory

New components required for this phase:

### 1. `components/HASettingsSection.tsx`
Section within SettingsClient for HA configuration. Contains:
- URL input (text, placeholder: `http://homeassistant.local:8123`)
- Token input (password type, placeholder: shown as bullets; shows "Token saved" hint when `haTokenSet` is true and input is empty)
- "Test Connection" button (accent color, full-width on mobile)
- Test result inline: success message OR error message below the button
- "Save" button (accent color, disabled when no dirty fields)

### 2. `components/HAEntityPicker.tsx`
Modal/drawer for entity selection after successful Test Connection:
- Search input at top (44px height, full-width)
- Entity list grouped by domain (domain name as section header in stone-500 label style)
- Each row: checkbox + friendly_name + entity_id in subdued text + domain badge
- Selection count indicator: "N of 10 selected" in amber when any selected
- "Save Selection" button (accent, disabled when nothing selected)
- Max 10 selection enforcement: disable unchecked rows when 10 are selected

### 3. `components/CampsiteCard.tsx`
Dashboard card (visible only when `haEntityIds` is non-empty):
- Card header: "Campsite" label + last-updated timestamp (right-aligned)
- Entity rows: icon (domain-appropriate Lucide icon) + friendly_name + state value + unit
- Unavailable/unknown state: render `—` in stone-400 color
- Stale state banner: amber background strip "Offline — last updated X ago"
- No-cache offline state: "Offline — no data yet" centered in card
- Loading skeleton: pulsing rows (use `.skeleton` CSS class from globals.css) — one row per expected entity count
- "Full View" link at card bottom right → `/campsite` (accent color chevron + text)

### 4. `app/campsite/page.tsx` + `components/CampsitePageClient.tsx`
Fullscreen tablet dashboard, dark mode forced:
- 2-column grid on tablet (768px+), 1-column on phone
- Each sensor card: friendly_name (14px label top), state value (28px/semibold center), unit (14px muted below)
- Domain icon top-right of each card (24px, stone-400)
- Status bar fixed at bottom: spinning `RefreshCw` icon when polling + "Last updated Xs ago" OR "Offline" in red
- No nav chrome — minimal layout, no AppShell
- Back link top-left: `← Outland OS` (14px, stone-400) linking to `/`

### 5. `components/HASnapshotSection.tsx`
Read-only section in trip prep:
- Section header: "Campsite Sensors" (same style as other prep section headers)
- Entity rows identical to CampsiteCard rows but without polling or auto-refresh
- Fetched once on page load, no refresh button
- Stale/offline state same as CampsiteCard pattern

## Interaction Contracts

### Settings — HA Configuration Section

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Type in URL or Token field | Keystroke | Fields become dirty; Save button becomes enabled |
| Click "Test Connection" | Click/tap | Button shows loading state ("Testing…"), disabled; result appears inline below button |
| Test succeeds | API response | Green checkmark + "Connected — N entities found" + "Configure Entities" link appears |
| Test fails | API response | Red X + error string (e.g., "Could not reach Home Assistant — check URL and token") |
| Click "Configure Entities" | Click | HAEntityPicker modal opens |
| Click "Save" | Click | Fields save, button shows "Saved" for 2s, token field clears and shows "Token saved" hint |
| Click "Save Selection" in picker | Click | Modal closes, Campsite card appears on dashboard within next poll |

### Dashboard Campsite Card

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Page load with entities configured | Mount | Skeleton appears immediately, first poll fires, skeleton replaced by data |
| Poll succeeds | 30s interval | Entity values update in-place, timestamp updates |
| Poll fails (HA unreachable) | Fetch error | Stale data stays visible, amber "Offline — last updated X ago" banner appears |
| No cache + offline | Fetch error, no cache | "Offline — no data yet" message shown |
| Click "Full View" | Click | Navigate to `/campsite` |

### /campsite Route

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Page load | Mount | Dark background renders immediately (CSS class forced), data fetches, cards populate |
| Auto-refresh | 30s interval | Values update silently, status bar shows "Last updated 0s ago" then counts up |
| Offline | Fetch fail | Status bar turns red "Offline", cached values remain, no spinner |
| Click back link | Click | Navigate to `/` |

### Entity Picker

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Type in search | Keystroke | Entity list filters in real-time (client-side, no API call) |
| Check entity | Click checkbox | Row shows checked state, selection count increments |
| Reach 10 entities | 10th selection | All unchecked rows disabled and visually muted (opacity 50%) |
| Uncheck entity | Click checked checkbox | Row unchecks, unchecked rows re-enable |
| Click "Save Selection" | Click | POST /api/settings with new haEntityIds, modal closes |

## States Required

Every new component must handle these states explicitly:

### CampsiteCard + /campsite route

| State | Condition | Visual |
|-------|-----------|--------|
| Hidden | `haEntityIds` empty or null | Component not rendered |
| Loading | First mount, before first poll response | Skeleton rows (`.skeleton` class) matching entity count |
| Live | Poll succeeded | Entity rows with fresh data, green indicator dot |
| Stale | Poll failed, cache exists | Entity rows from cache, amber "Offline — last updated X ago" banner |
| Dead | Poll failed, no cache | "Offline — no data yet" centered message, no entity rows |
| Unavailable entity | `state === 'unavailable'` or `state === 'unknown'` | Display "—" in stone-400; unit hidden |

### Test Connection button

| State | Visual |
|-------|--------|
| Idle (not configured) | Disabled, stone border style |
| Idle (configured) | Enabled, accent color |
| Loading | Disabled, "Testing…" label, spinner icon |
| Success | Result text in emerald below button |
| Error | Result text in red below button |

### Entity Picker

| State | Visual |
|-------|--------|
| Loading entities | Skeleton list |
| Populated | Domain-grouped list |
| Searching | Filtered list, unmatched rows hidden |
| Max reached | Unchecked rows at 50% opacity, pointer-events none |
| Saving | "Save Selection" button disabled, shows "Saving…" |

## Layout Contracts

### Settings HA Section

Follows existing SettingsClient section pattern: bordered Card component, section title as h3 (20px/600), form fields stacked vertically with 16px gap, CTA buttons right-aligned on desktop / full-width on mobile (under 640px).

### Dashboard Campsite Card

Follows existing dashboard card pattern from `DashboardClient.tsx`: `rounded-xl` border, `p-4` padding, stone surface background. Entity rows use `flex` with `justify-between`, 8px vertical gap between rows.

### /campsite Route Layout

```
Full viewport height, dark background (stone-950)
├── Top bar (48px): back link left, "Campsite" heading center, last-updated right
├── Entity card grid (flex-grow):
│   ├── Mobile (< 768px): 1 column
│   ├── Tablet (>= 768px): 2 columns, gap-6
│   └── Each card: rounded-xl, stone-900 bg, stone-700 border, p-6
│       ├── Domain icon top-right (24px, stone-400)
│       ├── Friendly name (14px, stone-400)
│       ├── State value (28px/600, stone-50)
│       └── Unit (14px, stone-500)
└── Status bar (40px fixed bottom): centered, RefreshCw icon + status text
```

No AppShell, no BottomNav on /campsite. Standalone layout.
