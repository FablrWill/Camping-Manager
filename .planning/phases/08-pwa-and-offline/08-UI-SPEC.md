## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Offline banner height: 36px fixed (slim single-line bar — not a multiple of 4 for layout reasons, but padding within it uses 8px vertical)
- "Leaving Now" button: minimum touch target 52px tall (lg Button size `py-3` = 24px padding top+bottom + 20px text = ~44px; override to 52px to exceed iOS 44pt minimum)
- iOS safe area bottom: `env(safe-area-inset-bottom)` applied via existing `.safe-bottom` class — not a spacing token

## Typography

Inherits existing system type scale. No new type sizes introduced.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Offline banner text, cache status messages, install banner body copy |
| Label | 12px (text-xs) | 400 (regular) | 1.4 | Snapshot age stamp, progress step labels, tile coverage notice |
| Heading | 18px (text-lg) | 700 (bold) | 1.2 | Progress overlay section headers ("Caching your trip...") |
| Display | 20px (text-xl) | 700 (bold) | 1.2 | "Leaving Now" button label |

Note: Label text at 12px is visually distinct from 14px body text through size alone — no weight differentiation needed. "Leaving Now" button uses `text-base` (16px) at Button lg size per existing `sizeStyles.lg`. Display row describes visual weight hierarchy, not a new size.

## Component Inventory — New Components This Phase

### 1. OfflineBanner

**Placement:** Rendered inside `AppShell.tsx`, between `<TopHeader />` and `<main>`. Fixed slim bar, always visible when offline. Hidden when online.

**Anatomy:**
```
[━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━]
[  WifiOff icon  Offline · snapshot from 2h ago  ]
[━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━]
```
- Height: 36px
- Background: stone-800 (light), stone-900 (dark)
- Left border stripe: 3px solid amber-500
- Icon: `WifiOff` (Lucide, 14px)
- Text: "Offline" (font-medium) + " · snapshot from {relative time}" (text-stone-400)
- Relative time format: "just now", "5m ago", "2h ago", "3 days ago"
- Persists across page navigation (lives in AppShell)
- Slides down 36px from top when it appears (CSS transition 200ms)
- Main content area gains 36px top offset when banner is visible

**Interaction:** No dismiss. Disappears automatically when connectivity returns.

**Connectivity detection:** `navigator.onLine` + `window` `online`/`offline` events.

**Two states:**
- Default: large amber primary button "Leave Now — Cache Trip Data"
- Already-cached: secondary variant "Update Cache" with timestamp "Cached 3h ago"

- Icon: `Tent` (Lucide, 20px, left-aligned)

### 4. InstallBanner

**Placement:** Rendered below `OfflineBanner` (or below `TopHeader` if no offline banner), above `<main>`. Dismissible.

**When shown:** First visit only. Not shown if app is already in standalone mode (`window.matchMedia('(display-mode: standalone)').matches`). Persisted via `localStorage` key `outland_install_dismissed`.

**Two paths:**

**iOS (Safari):**
```
┌─────────────────────────────────────────────────┐
│  📱 Add to Home Screen for offline access        │
│  Tap [ Share ↑ ] then "Add to Home Screen"       │
│                                            [✕]   │
└─────────────────────────────────────────────────┘
```
- User agent detect: iOS + Safari (no `BeforeInstallPromptEvent` support)
- No "Install" button — shows inline instruction with Share icon (BoxArrowUp equivalent: `Share` from Lucide or inline SVG)

**Android/Chrome:**
```
┌─────────────────────────────────────────────────┐
│  📱 Add Outland OS to your home screen           │
│  Access your trip data offline                   │
│  [  Install App  ]                    [✕]        │
└─────────────────────────────────────────────────┘
```
- Uses `BeforeInstallPromptEvent` — "Install App" button triggers `prompt()`
- Button: primary, sm size

**Styling:**
- Background: stone-800 (always dark — matches BottomNav aesthetic)
- Text: stone-200 title (text-sm font-medium), stone-400 subtitle (text-xs)
- Dismiss button: `X` Lucide, 16px, stone-400, top-right, `aria-label="Dismiss install banner"`
- Padding: 12px horizontal, 8px vertical
- Border-bottom: 1px stone-700

## Interaction Contracts

### "Leave Now" tap
1. Tap button — no confirmation
2. Full-screen `CachingProgressOverlay` appears immediately (200ms fade-in)
3. Steps execute sequentially, each row transitions: pending → in-progress → complete
4. Progress bar animates smoothly from 0 to 100%
5. "Done — you're ready to go" button appears after final step
6. Auto-dismiss after 1.5s if all steps succeeded, OR user taps "Done" button
7. If a step fails: mark with AlertCircle + inline error text. Do NOT block remaining steps. "Done" button still appears.

### Offline state transitions
- **Coming online:** OfflineBanner slides up (animates out, 200ms). Background re-sync silently triggers. Banner timestamp updates on next offline event.
- **Going offline:** OfflineBanner slides down (animates in, 200ms). No sound. No modal. Banner persists until online.
- **Navigation while offline:** Banner re-renders on each page. Non-cached pages show an EmptyState (existing component pattern) with copy: "No offline data" / "Connect to load this page."

### AI features while offline (D-08)
- Chat: shows EmptyState — "Chat requires a connection. Your cached trip data is available offline."
- Generate buttons (packing list, meal plan, checklist): Button disabled, tooltip/caption: "Connect to generate"
- Already-generated content: shows normally from cache

### Offline writes (D-19)
- Departure checklist check-offs: **read-only while offline**. Checkboxes render as disabled. Caption below checklist: "Check-offs sync when you reconnect."
- Rationale: simplest and most reliable for single-user local-first app. No queue needed.
