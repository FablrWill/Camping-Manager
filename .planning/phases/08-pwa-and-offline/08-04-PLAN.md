<objective>
Add map tile passive caching via service worker, offline UI states for AI features and checklist, and staleness warning.

Purpose: Map tiles are available offline from normal browsing (OFF-04), and the app gracefully handles offline state across all interactive features (OFF-02, OFF-03).
Output: Updated SpotMap with crossOrigin, SW tile caching, offline states for checklist/AI, staleness warnings

**Design deviation from D-12/D-13/D-14 (tile caching):** CONTEXT.md decisions D-12, D-13, and D-14 specified proactive tile pre-fetching at "Leaving Now" time (cache viewport + 2 zoom levels, tiles around saved locations within 20 miles). After research (08-RESEARCH.md), this was changed to **passive intercept-only caching** because OpenStreetMap's Tile Usage Policy explicitly prohibits bulk pre-fetching. The service worker passively caches tiles as the user views them during normal browsing, which provides good coverage of frequently visited areas while respecting OSM's terms. If Will wants proactive region downloads, the recommended path is deep-linking to a dedicated app like Gaia GPS (already noted as out-of-scope in REQUIREMENTS.md).
</objective>
