> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

| Property | Value | Source |

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (gap-1), inline badge padding |
| sm | 8px | Compact element spacing (gap-2, p-2) |
| md | 16px | Default element padding (p-4, gap-4) |
| lg | 24px | Section padding (p-6, gap-6) |
| xl | 32px | Layout gaps (gap-8) |
| 2xl | 48px | Major section breaks |

- Voice record button: minimum 64px × 64px touch target (mobile, thumb-primary)

## Typography

Pre-populated from existing component audit (Card.tsx, ChatBubble.tsx, Button.tsx, Badge.tsx).

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 (leading-relaxed) | `text-base leading-relaxed` |
| Label / UI text | 14px | 400 (regular) | 1.4 | `text-sm` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |
| Caption / metadata | 12px | 400 (regular) | 1.3 | `text-xs` |

**Weights in use:** regular (400) and semibold (600) only. No bold (700) in this phase.

**Phase 5 specific:**
- Recommendation card spot name: 16px semibold (`text-base font-semibold`)
- Recommendation card distance/meta line: 14px regular (`text-sm`)
- Voice debrief section headers (what worked / what didn't): 14px semibold (`text-sm font-semibold`)
- Extracted insight body text: 14px regular (`text-sm`)
- Recording timer counter: 20px regular (`text-xl`) — monospaced feel via `tabular-nums`

## Color

Pre-populated from `app/globals.css` design tokens. Stone/amber palette with full dark mode.

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#fafaf9` (stone-50) | `#0c0a09` (stone-950) | Page background, chat thread background |
| Secondary (30%) | `#ffffff` (white) | `#1c1917` (stone-900) | Cards (recommendation cards, insight cards), nav bar, chat bubbles (assistant) |
| Surface alt | `#f5f5f4` (stone-100) | `#292524` (stone-800) | Input backgrounds, skeleton loaders, collapsed sections |
| Accent (10%) | `#d97706` (amber-600) | `#f59e0b` (amber-500) | See reserved list below |
| Destructive | `#dc2626` (red-600) | `#ef4444` (red-500) | Destructive actions only — see below |
| Success | `#059669` (emerald-600) | `#10b981` (emerald-500) | Insight applied confirmation state, saved location badge |

**Accent (`--color-accent`) reserved for:**
- User chat bubble background
- Primary CTA buttons (Save Spot, Send, Apply Insight)
- Voice record button background (active/idle state)
- Active recording pulse ring border
- Send button in ChatInput
- Recommendation card Save button
- Accent Badge variant (`badge-accent`)

**Accent NOT used for:**
- Text links (use `--color-text-primary` with underline)
- Informational badges (use stone/muted variants)
- Navigation icons (use stone with amber for active tab only)

**Destructive color reserved for:**
- Insight discard confirmation button ("Discard all insights")
- Delete confirmation buttons (inheriting existing `deleteConfirmTool` pattern)
- Recording cancel when insights have been extracted

**Voice recording states — color contract:**
- Idle: amber-600 background, white icon
- Recording: red-600 pulsing ring, amber-600 background → transitions to solid red-600 background during active recording
- Processing/transcribing: stone-300 disabled background, spinner
- Complete: emerald-600 checkmark flash (200ms), then back to idle

## Component Inventory

New components required for Phase 5. All reuse existing token set.

### RecommendationCard

Renders inside the chat thread as a rich location result card. Reuses `Card` component from `components/ui/Card.tsx`.

```
[Card — hoverable=false, padding='md']
  [spot name — text-base font-semibold text-stone-900 dark:text-stone-50]
  [distance badge — text-xs, Badge variant='muted'] [rating stars if saved — text-amber-500]
  [description — text-sm text-stone-500 dark:text-stone-400, 2-line clamp]
  [source badge — "Your spot" (accent) or "From knowledge base" (muted)]
  [Save button — Button variant='primary' size='sm', full-width, disabled if already saved]
```

- Multiple recommendation cards stack vertically in the chat thread with `gap-3`
- Cards appear below the assistant's text response in the same assistant bubble container
- "Already saved" state: Save button disabled, text changes to "Saved", variant='secondary'
- Save button triggers navigation to location edit form after POST — not a toast

### VoiceDebriefButton

Floating entry point on the Trips page — positioned as a secondary action on completed trip cards.

```
[Trip card footer row]
  [...existing trip actions]
  [Mic icon button — ghost variant, size='sm', 16px Lucide Mic icon]
  aria-label="Record trip debrief"
```

No dedicated tab. Entry point is the trip card's action row. Tapping opens the voice debrief modal.

### VoiceRecordModal

Full-screen bottom sheet (reuses `animate-slide-up` class from globals.css).

**Idle state:**
```
[Modal overlay]
  [Bottom sheet — bg-white dark:bg-stone-900, rounded-t-2xl, p-6]
    [Title — "Trip Debrief" — text-xl font-semibold]
    [Subtitle — "Record what happened. Claude will extract the notes." — text-sm text-stone-500]
    [Record button — 64×64px circle, amber-600 bg, white Mic icon 24px]
    [Cancel — ghost button below]
```

**Recording state:**
```
    [Timer — "0:00" counting up, text-xl tabular-nums text-stone-900 dark:text-stone-50]
    [Animated pulse ring — red-600, 80px outer, 64px inner record button]
    [Record button — red-600 bg, white Square icon 24px (tap to stop)]
    [Waveform placeholder — 3 animated bars, stone-400, height varies 8-32px]
    [Cancel — ghost button, "Cancel recording"]
```

**Processing state (transcribing + extracting):**
```
    [Spinner — stone-400, 32px]
    ["Transcribing..." then "Extracting insights..." — text-sm text-stone-500]
    [No cancel during processing — button disabled]
```

### InsightsReviewSheet

Slides up after extraction completes. Shows structured insights for manual review before applying.

```
[Bottom sheet — full screen height, scrollable]
  [Header — "Review Insights" — text-xl font-semibold]
  [Close X — top-right, ghost icon button]

  [Section: What Worked — text-sm font-semibold text-emerald-700 dark:text-emerald-400]
    [InsightRow × N — checkbox + text-sm body, each independently appliable]

  [Section: What Didn't — text-sm font-semibold text-red-600 dark:text-red-400]
    [InsightRow × N]

  [Section: Gear Feedback — text-sm font-semibold text-stone-700 dark:text-stone-300]
    [InsightRow × N — each shows which gear item it targets]

  [Section: Spot Rating — text-sm font-semibold text-stone-700 dark:text-stone-300]
    [Star rating display — 5 stars, amber-500 filled, tappable to adjust]

  [Footer — sticky]
    [Button "Apply Selected" — primary, full-width]
    [Button "Discard All" — danger ghost, full-width, requires confirmation]
```

**InsightRow:**
```
[checkbox — amber-600 when checked] [insight text — text-sm] [target chip — Badge muted, e.g. "Gear: Rain Fly"]
```

**Apply confirmation pattern:** Inline — no modal. On tap "Apply Selected", button shows spinner + "Applying..." then "Done" with emerald checkmark. No separate confirmation dialog for apply (non-destructive).

**Discard confirmation pattern:** Inline — the "Discard All" button expands to show "Tap again to discard all insights. This can't be undone." with a second danger button inline below. No modal. This mirrors the `deleteConfirmTool` chat pattern.

## Interaction Patterns

### Recommendation card flow
1. User types "find me a spot near [location] this weekend" in chat
2. Agent streams text response, then renders 1–5 RecommendationCard components below the bubble
3. User taps "Save Spot" on a card
4. POST to `/api/locations` — button shows spinner "Saving..."
5. On success: button changes to "Saved" (disabled, secondary variant) and app navigates to `/spots` or location edit form
6. On failure: inline error below button, button resets

### Voice recording flow
1. User taps mic icon on a trip card → VoiceRecordModal slides up from bottom (`animate-slide-up`)
2. User taps record button → recording starts, timer counts up, waveform animates
3. User taps stop → audio captured, modal transitions to processing state
4. Transcription completes → extraction runs → InsightsReviewSheet slides up over the modal
5. User reviews, checks/unchecks insights, taps "Apply Selected"
6. Each selected insight is applied (gear notes updated, location rating set, trip notes appended)
7. Sheet shows "Done" confirmation, auto-closes after 1.5s
8. VoiceRecordModal also closes

### Insight application — confirm before overwrite
- If applying a gear note update and gear already has notes: append, do not overwrite
- If applying a location rating and location already has a rating: show current rating inline with "Replace [3 stars] with [4 stars]?" as a single checkbox before applying
- This follows CONTEXT.md D-06: "confirm before committing" for data writes

## Accessibility

- Voice record button: `aria-label="Start recording"` / `aria-label="Stop recording"` (state-dependent)
- Recording in progress: `aria-live="polite"` region announcing "Recording: 0:23"
- Insights review: checkboxes have `aria-label="{insight text}"`
- Recommendation cards: `role="article"` with `aria-label="{spot name} recommendation"`
- Save Spot button: `aria-disabled="true"` with `aria-label="Already saved"` when in saved state
- Minimum tap targets: 44px × 44px for all interactive elements, 64px × 64px for voice record button
