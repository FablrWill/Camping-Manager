   Default to `null` for all recommendations initially (saved and knowledge_base).

3. After building the sorted results list (line ~107) but before the `return JSON.stringify(...)`, fetch weather for recommendations that have coordinates. Use `Promise.allSettled` to fetch in parallel and tolerate failures:

   ```typescript
   // --- Source 3: Weather forecasts (best-effort enrichment) ---
   const weatherPromises = sorted.map(async (rec) => {
     if (!rec.coordinates) return;
     try {
       const raw = await executeGetWeather({
         latitude: rec.coordinates.lat,
         longitude: rec.coordinates.lon,
         days: 3,
       });
       const parsed = JSON.parse(raw);
       if (parsed.forecast) {
         rec.weatherSummary = parsed.forecast.slice(0, 3);
       }
     } catch {
       // Best-effort — leave weatherSummary as null
     }
   });
   await Promise.allSettled(weatherPromises);
   ```

4. Keep `days: 3` to limit API calls and output size. Three days of forecast is enough context for the agent to mention weather conditions.

5. Do NOT change the tool schema (input_schema) or the function signature. The output JSON just gets a new optional field per recommendation.

Key constraint: Weather fetch is best-effort. If Open-Meteo is down or a fetch fails, the recommendation still appears with `weatherSummary: null`. The try/catch inside the map callback plus `Promise.allSettled` ensures no single failure breaks the batch.
