      ```json
        "locationId": null,
        "notes": "...",
        "bringingDog": false
      }
      ```

    Export `webSearchCampsitesTool` (Tool schema) and `executeWebSearchCampsites` (executor function).

    Tool schema:
    - name: `'web_search_campsites'`
    - description: `'Search the web for campsite info, availability, road conditions, or permit requirements for a destination.'`
    - input_schema: object with required `query` (string) property

    Executor function `executeWebSearchCampsites(input: { query: string }): Promise<string>`:
    - Use `fetch` to call the Anthropic messages API directly with a single user message containing the search query and instructions to search the web and summarize findings
    - Use `process.env.ANTHROPIC_API_KEY` for auth
    - Set a 10-second `AbortController` timeout on the fetch
    - On timeout or error, return `'Web search timed out or failed. Suggest the user check Recreation.gov or Google for campsite info.'`
    - On success, return the text content from the response (trimmed to first 2000 chars to keep tool results manageable)
    - Model: `'claude-sonnet-4-20250514'` with `max_tokens: 1024`
    - Alternative simpler approach: If the Anthropic SDK `@anthropic-ai/sdk` version supports `client.messages.create` with `tools` containing a `web_search` type tool, use that instead. Check by importing Anthropic and inspecting available tool types. If not available, use the direct fetch approach above.
