<task type="auto">
  <name>Task 3: Configure tailscale serve for persistent HTTPS proxy</name>
  <files></files>
  <read_first>
    .planning/phases/15-remote-access-go-live/15-RESEARCH.md
  </read_first>
  <action>
    Run `tailscale serve` with the `--bg` flag to create a persistent HTTPS proxy from the MagicDNS hostname to localhost:3000. The `--bg` flag stores the configuration inside the Tailscale daemon state — no launchd plist, cron, or deploy script change needed.

    Steps:
    1. Run: `tailscale serve --bg --https=443 http://localhost:3000`
    2. Verify: `tailscale serve status`
       Expected output:
       ```
       https://[hostname].ts.net
       |-- / proxy http://127.0.0.1:3000
       ```
    3. Test from Mac mini itself: `curl -s -o /dev/null -w "%{http_code}" https://[hostname].ts.net`
       Expected: 200 (or 302 if app redirects)

    Per D-01: Use `tailscale serve` for HTTPS. No reverse proxy (Caddy, Nginx) needed.
    Per D-02: Next.js stays on plain HTTP at localhost:3000. HTTPS termination is 100% Tailscale's job.

    CRITICAL: Use `--bg` flag. Without it, the serve config is lost when the terminal closes or Mac mini reboots.

    Persistence decision (Claude's discretion per CONTEXT.md): Keep `tailscale serve` as a one-time setup, NOT in the deploy script. The `--bg` flag makes serve self-persisting inside the Tailscale daemon. The deploy script (git pull, build, pm2 restart) does not need to touch Tailscale.
  </action>
  <verify>
    <automated>tailscale serve status</automated>
  </verify>
  <acceptance_criteria>
    - `tailscale serve status` output contains "proxy http://127.0.0.1:3000"
    - `tailscale serve status` output contains "https://" and ".ts.net"
    - `curl https://[hostname].ts.net` returns HTTP 200 or 302 (app is reachable via HTTPS)
  </acceptance_criteria>
  <done>HTTPS proxy is active and persistent. The app is reachable at https://[hostname].ts.net from any device on the tailnet (ACCESS-03 and ACCESS-04 complete)</done>
