    **Prerequisites section:**
    - Node.js 20 LTS installed
    - git installed
    - Apple ID signed into Messages app

    **Step 1 -- Create data directory (DEPLOY-02):**
    ```bash
    sudo mkdir -p /data/outland
    sudo chown -R $(whoami) /data/outland
    mkdir -p /data/outland/{photos,backups,logs}
    ```

    **Step 2 -- Clone the repo:**
    ```bash
    mkdir -p ~/outland
    git clone <repo-url> ~/outland
    cd ~/outland
    ```

    **Step 3 -- Create production .env:**
    - Copy from .env.example: `cp .env.example .env`
    - Edit to set production values:
      - `DATABASE_URL="file:/data/outland/db.sqlite"`
      - `PHOTOS_DIR=/data/outland/photos`
      - `DATA_DIR=/data/outland`
      - `NOTIFY_NUMBER=+1XXXXXXXXXX` (Will's actual phone number)
      - Uncomment and fill in: ANTHROPIC_API_KEY, OPENAI_API_KEY, VOYAGE_API_KEY, GMAIL_USER, GMAIL_APP_PASSWORD

    **Step 4 -- Install PM2 globally (DEPLOY-04):**
    ```bash
    npm install -g pm2
    ```

    **Step 5 -- Migrate existing photos (REQUIRED per D-09):**

    If photos exist in the dev environment, they MUST be copied to the persistent storage location before first deploy. New uploads will go to `/data/outland/photos/` but existing photos referenced in the database need to be there too.

    ```bash
    # From MacBook, copy existing photos to Mac mini:
    rsync -avz public/photos/ mac-mini:/data/outland/photos/
    ```

    If no photos exist yet (fresh install), skip this step -- the deploy script creates the directory.

    **Step 6 -- First deploy:**
    ```bash
    chmod +x deploy.sh scripts/*.sh
    ./deploy.sh
    ```

    **Step 7 -- Set up PM2 boot persistence (DEPLOY-05):**
    ```bash
    pm2 startup
    # Run the sudo command it outputs (copy-paste the exact line)
    pm2 save
    ```
    Note: This uses launchd on macOS (not systemd). The generated plist lives at `~/Library/LaunchAgents/`.

    **Step 8 -- Set up crontab for backups and watchdog (DEPLOY-07, D-03, D-07):**
    ```bash
    crontab -e
    ```
    Add these two lines:
    ```
    0 2 * * * ~/outland/scripts/outland-backup.sh >> /data/outland/logs/backup.log 2>&1
    */5 * * * * ~/outland/scripts/outland-watchdog.sh 2>/dev/null
    ```

    **Step 9 -- Test iMessage notifications:**
    ```bash
    osascript -e 'tell application "Messages" to send "Outland setup test" to buddy "+1XXXXXXXXXX"'
    ```

    **Step 10 -- Verify everything works:**
    - `curl http://localhost:3000/api/health` returns JSON with `status: 'ok'`
    - `pm2 list` shows `outland` with status `online`
    - `pm2 show outland` shows fork mode, correct log paths
    - Open a photo URL in the browser to confirm existing photos resolve

    **Subsequent deploys section:**
    ```bash
    ssh mac-mini 'cd ~/outland && ./deploy.sh'
    ```

    **Troubleshooting section** with the known pitfalls from research:
    - App not accessible from Tailscale: check HOSTNAME binding
    - Photos showing as broken: check photo symlink and that rsync migration was done
    - PM2 not starting after reboot: run `pm2 save` again
    - Backup iMessage not arriving: check Messages app is signed in
