## Spacing Scale

Declared values (must be multiples of 4 — confirmed from existing card patterns in TripPrepClient.tsx):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding between name and distance |
| sm | 8px | Compact element spacing within category rows (`gap-2`) |
| md | 16px | Card internal padding (`p-4` = 16px) |
| lg | 24px | Section padding in TripPrepClient outer wrapper (`py-6` = 24px) |
| xl | 32px | Layout gaps (not used in this card) |
| 2xl | 48px | Major section breaks (not used in this card) |
| 3xl | 64px | Page-level spacing (not used in this card) |

Exceptions:
- Touch targets for the card header: minimum 44px tall (natural with `p-4` padding + text)
- Separator dot between stops in the same category row: `mx-1` (4px) — grid-aligned minimum, matches spacing grid

**Source:** TripPrepClient.tsx card shell `p-4`, WeatherCard.tsx inner spacing patterns.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Card heading | 14px (text-sm) | 600 (font-semibold) | 1.25 | `text-sm font-semibold` |
| Stop name | 14px (text-sm) | 400 (font-normal) | 1.5 | `text-sm` |
| Distance label | 12px (text-xs) | 400 (font-normal) | 1.4 | `text-xs` |
| Category label | 12px (text-xs) | 600 (font-semibold) | 1.25 | `text-xs font-semibold uppercase` |

Only 2 font weights used: 400 (regular) and 600 (semibold). Matches existing WeatherCard pattern. Category labels use `font-semibold` (same weight as the card heading) — their visual distinction comes from `uppercase` and smaller size (`text-xs`), not a third weight.

**Source:** WeatherCard.tsx heading uses `text-sm font-semibold`; day labels use `text-xs font-medium uppercase`; data values use `text-sm`; secondary labels use `text-xs`.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Page load with location set | Card mounts, triggers `useEffect`, shows skeleton immediately, replaces with data when fetch resolves |
| Page load with no location | Card does not render — silent omit (D-01) |
| Overpass fetch succeeds | Skeleton replaced with category rows; "None found" shown for any category with 0 results |
| Overpass fetch fails (network/500) | Skeleton replaced with error state card (red surface, error message, no retry button) |
| Card body | Read-only — no interactive elements within the card |

No expand/collapse toggle. Unlike WeatherCard, the Fuel card does not have an expand state — all results (max 2 per category) are always visible.

**Source:** CONTEXT.md D-01, D-04, D-10; WeatherCard.tsx for comparison (has expand; Fuel does not).

## Layout Within TripPrepClient

Insertion order in `max-w-4xl mx-auto px-4 py-6 space-y-4`:

1. Back navigation link
2. Trip header card
3. Error state (if prep fetch failed)
4. Loading skeleton (while prep fetch in flight)
5. WeatherCard (when prep fetch complete, if location set)
6. **Fuel & Last Stops card** ← NEW, here
7. PREP_SECTIONS.map() block
8. PackingList
9. MealPlan
10. PowerBudget

The card is rendered outside the `PREP_SECTIONS.map()` loop and is NOT in the `PREP_SECTIONS` constant.

**Source:** CONTEXT.md D-09; TripPrepClient.tsx structure confirmed in research.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Loading skeleton | Wrap in `role="status" aria-label="Loading nearby stops"` |
| Category labels | Use `<h4>` or `<p className="... font-medium uppercase">` — consistent with WeatherCard label pattern |
| Distance values | Plain text — screen readers read "{name} — {distance} mi" naturally |
| Error state | Wrap in `role="alert"` so screen readers announce it immediately |
| No interactive elements | No keyboard focus management needed |
