## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | stone-50 / stone-950 dark | Page background — inherited |
| Secondary (30%) | white / stone-900 dark | Card surfaces — inherited |
| Accent (10%) | amber-600 / amber-500 dark (`#d97706` / `#f59e0b`) | "Good for stars" indicator badge only |
| Destructive | red-600 / red-500 dark | — (no destructive actions in this phase) |

**AstroCard tint:** Use `bg-indigo-50 dark:bg-indigo-950/30` with `border-indigo-200 dark:border-indigo-800` — distinct from weather (sky-50/sky-200) and fuel card, signals astronomical context. Loading skeleton uses `bg-indigo-200 dark:bg-indigo-800` placeholder blocks.

**Accent reserved for:**
- The "Good for stars" / "Poor for stars" status badge when moon fraction < 25% (good) — amber tint: `bg-amber-50 dark:bg-amber-950/30 text-amber-700 dark:text-amber-300`
- The "Poor for stars" badge when moon fraction >= 25% — muted: `bg-stone-100 dark:bg-stone-800 text-stone-500 dark:text-stone-400`
- The "Check light pollution" external link — uses amber text (`text-amber-600 dark:text-amber-400`) with ExternalLink icon

**Source:** `globals.css` token definitions; `WeatherCard.tsx` sky tint pattern (mirrored with indigo for astro); `Button.tsx` amber accent values.

## Component Inventory

### New: `AstroCard.tsx`

Location: `components/AstroCard.tsx`

Structure mirrors `WeatherCard.tsx`:

```
AstroCard
  ├── LoadingSkeleton (indigo tint, 3-column animate-pulse)
  ├── Error state (red tint, inline message)
  ├── No-location state (inline prompt: "Add a location to see sunrise/sunset times")
  ├── Card container (indigo-50/indigo-950/30, rounded-xl, border)
  │   ├── Header row
  │   │   ├── Title: "🌙 Night Sky" (text-sm font-semibold)
  │   │   ├── Subtitle: location name · date range (text-xs text-stone-500)
  │   │   └── Expand/collapse chevron (ChevronDown/ChevronUp, ghost interaction)
  │   ├── Per-night columns (grid, 3 columns max, text-center)
  │   │   ├── NightBlock (per NightAstro entry)
  │   │   │   ├── Date label: "Mon", "Tue" (text-xs uppercase font-medium text-stone-500)
  │   │   │   ├── Moon emoji: 🌑🌒🌓🌔🌕🌖🌗🌘 (text-2xl leading-none)
  │   │   │   ├── Moon label: "Full Moon" (text-xs text-stone-500)
  │   │   │   └── [expanded] Sunrise/sunset row (text-xs, Sunrise + Sunset icons)
  │   ├── Stars quality badge: "Good for stars" / "Poor for stars" (below grid, full-width)
  │   └── [coords available] Bortle link: "Check light pollution →" (text-xs, ExternalLink icon)
```

**Props:**
```typescript
interface AstroCardProps {
  nights: NightAstro[]
  loading?: boolean
  error?: string | null
  locationName?: string
  dateRange?: string
  bortleLink?: string
}
```

**Collapsed state:** Shows moon emoji + moon label per night column only. No sunrise/sunset.
**Expanded state:** Adds sunrise/sunset row per night column below the moon label.
**Expand trigger:** Chevron button in card header. Default state: collapsed.

### New: `lib/astro.ts`

Export shape (UI-facing — data contract):
```typescript
export interface NightAstro {
  date: string          // YYYY-MM-DD
  moonPhase: number     // 0.0–1.0
  moonFraction: number  // 0.0–1.0
  moonLabel: string     // "Full Moon", "New Moon", etc.
  moonEmoji: string     // "🌕", "🌑", etc.
  goodForStars: boolean // moonFraction < 0.25
  sunrise?: string      // "6:42 AM" — from DayForecast when available
  sunset?: string       // "7:58 PM" — from DayForecast when available
}

export interface TripAstroData {
  nights: NightAstro[]
  bortleLink?: string   // lightpollutionmap.info deep link when coords available
}
```

### Modified: `TripCard.tsx`

Add `astro?: TripAstroData` prop. Render `<AstroCard>` below `<WeatherCard>` inside the expanded trip detail section. No changes to `TripCard` collapsed state.

### Modified: `TripsClient.tsx`

Mirror the existing `weatherByTrip` / `weatherLoading` / `weatherErrors` state pattern with `astroByTrip` / `astroLoading` / `astroErrors`. Compute astro client-side (no API round-trip) using `suncalc` in a utility hook or inline in `useEffect` — pure math, no fetch needed.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| AstroCard collapsed | Shows moon emoji + label per night; "Good/Poor for stars" badge; Bortle link if coords available |
| AstroCard expanded | Adds sunrise + sunset per night column |
| Expand/collapse | Chevron button in card header; toggles expanded state; no animation required (consistent with WeatherCard) |
| Bortle link tap | Opens `https://www.lightpollutionmap.info/...` in new tab (`target="_blank" rel="noopener noreferrer"`) |
| No location on trip | AstroCard shows moon phase per night (from trip dates) but hides sunrise/sunset rows and Bortle link; shows inline note: "Add a location to see sunrise/sunset times" |
| No trip dates | AstroCard is not rendered — component requires at least one night |
| Loading | IndulgentSkeleton (3-column animate-pulse, indigo tint) |
| Error | Inline red-tinted error message (matches WeatherCard error pattern) |

## Implementation Constraints (from CLAUDE.md + RESEARCH.md)

- No `alert()` — all error messages as inline state
- All React hooks: correct minimal dependency arrays
- Files: 200–400 lines typical, 800 max — `AstroCard.tsx` should stay under 200 lines
- Functions: <50 lines — split `NightBlock` into a sub-component like `DayBlock` in `WeatherCard`
- Immutable patterns — no in-place mutation of `nights` array
- No hardcoded values — moon phase thresholds in a named constant object (matches `getMoonPhaseLabel` pattern from RESEARCH.md)
- `suncalc` is pure CJS, safe in both API routes and client components
- Dark mode: every Tailwind class must have a `dark:` counterpart — follow WeatherCard exactly
