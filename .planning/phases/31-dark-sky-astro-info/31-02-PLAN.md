Add state (mirror weather pattern, near line 56-58):
```typescript
const [astroByTrip, setAstroByTrip] = useState<Record<string, TripAstroData>>({})
```

Since computeAstro is synchronous (pure computation, no fetch), compute it in a dedicated useEffect. **CRITICAL: Do NOT include `astroByTrip` in the dependency array — it is the state this effect writes via `setAstroByTrip`. Including it would cause infinite re-renders (CLAUDE.md rule: "never include state that the hook itself updates").**

Use a **batched single setState** (not forEach + multiple setters) and key the skip-guard on both trip.id AND whether weather data has arrived — so sunrise/sunset merges correctly after the async weather fetch completes:

```typescript
// Compute astro data for upcoming trips (pure computation, no API call)
useEffect(() => {
  const upcoming = trips.filter((t) => t.endDate >= now)
  if (upcoming.length === 0) return

  setAstroByTrip((prev) => {
    const next: Record<string, TripAstroData> = { ...prev }
    let changed = false

    upcoming.forEach((trip) => {
      const start = trip.startDate.split('T')[0]
      const end = trip.endDate.split('T')[0]
      const weatherData = weatherByTrip[trip.id]
      // Cache key: trip.id + whether weather data is present.
      // Re-compute when weather arrives so sunrise/sunset can merge.
      const hasWeather = !!weatherData?.days?.length
      const existingHasWeather = !!next[trip.id]?.nights[0]?.sunrise
      if (next[trip.id] && (existingHasWeather || !hasWeather)) return // already up-to-date

      next[trip.id] = computeAstro({
        startDate: start,
        endDate: end,
        lat: trip.location?.latitude ?? undefined,
        lon: trip.location?.longitude ?? undefined,
        weatherDays: weatherData?.days,
      })
      changed = true
    })

    return changed ? next : prev // return same reference if nothing changed — no re-render
  })
}, [trips, now, weatherByTrip])
```

Key details of this pattern:
- Dependency array is `[trips, now, weatherByTrip]` — only external inputs, NOT `astroByTrip`
- Single `setAstroByTrip` call (not one per trip) — batched update, minimal re-renders
- Guard logic: skip if already computed AND (weather already merged OR no weather available yet)
- Re-computes when `weatherByTrip` changes and the existing entry lacks sunrise/sunset — fixes the cache invalidation bug where sunrise/sunset never merged after async weather fetch

Pass astro prop to TripCard in both upcoming and past sections:
