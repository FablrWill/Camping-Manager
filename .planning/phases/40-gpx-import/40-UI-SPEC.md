## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (gap-2, py-1.5) |
| md | 16px | Default element spacing (p-3 = 12px, p-4 = 16px) |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Layer toggle pills: `px-2.5 py-1` (10px / 4px) — matches existing toggle button pattern exactly
- Import panel interior: `p-3` (12px) — matches existing GPX and Google Maps import panel pattern
- Trail list rows: `py-2 px-3` (8px / 12px) — compact list row within import panel

Source: Observed from `spots-client.tsx` lines 358-468.

## Color

|------|---------------|--------------|-------|

Accent reserved for:

## Component Inventory

All new UI elements in this phase extend existing patterns in `spots-client.tsx`. No new primitive components needed.

### Layer Toggle: "Trails" Pill

- Location: Layer toggle bar in controls bar (`flex gap-1 flex-wrap` div, lines 360-390)
- Pattern: Append `["trails", "🥾 Trails"]` to the existing `([...] as const).map()` array
- Active state: `bg-stone-800 text-white dark:bg-stone-200 dark:text-stone-900`
- Inactive state: `bg-stone-200 text-stone-500 dark:bg-stone-700 dark:text-stone-400`
- Size: `px-2.5 py-1 text-xs rounded-full font-medium transition-colors`
- Touch target: minimum 32px height at 4px vertical padding + 12px font = ~32px — acceptable for a map control bar

### GPX Import Panel Extension

The existing panel at lines 444-468 is extended to show a trails list below the file input and status message. The panel structure stays identical.

New section added inside the existing panel, below `{gpxMessage}`:

```
[divider: border-t border-stone-200 dark:border-stone-700 mt-3 pt-3]
[heading: "Saved Trails" — text-xs font-medium text-stone-500 dark:text-stone-400 uppercase tracking-wide mb-2]
[trail list — see Trail List Row below]
[empty state — see below]
```

### Trail List Row

- Layout: `flex items-center justify-between py-2`
- Left: trail name — `text-sm text-stone-700 dark:text-stone-300`
- Left sub: distance if available — `text-xs text-stone-500 dark:text-stone-400 ml-1` (e.g. "4.2 km")
- Right: delete button — ghost icon button, `text-stone-400 hover:text-red-600 dark:hover:text-red-400 transition-colors p-1 rounded`
- Separator between rows: none (tight list, panel is compact)
- Loading state: three skeleton rows `animate-pulse bg-stone-200 dark:bg-stone-700 h-4 rounded`

### Trail Map Popup

Triggered by clicking a trail polyline on the map (`layer.bindPopup()`).

Content:
```html
<div style="min-width: 140px; font-family: inherit;">
  <strong style="font-size: 14px; color: #1c1917;">{trail.name}</strong>
  {distanceKm ? `<div style="font-size: 12px; color: #78716c; margin-top: 2px;">{distanceKm.toFixed(1)} km</div>` : ''}
  <div style="font-size: 12px; color: #78716c; margin-top: 2px;">Imported {formattedDate}</div>
</div>
```

Note: Leaflet popups use inline styles (no Tailwind). Colors match `--color-text-primary` (#1c1917) and `--color-text-secondary` (#78716c) from the design token system.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (file picker button) | "Choose GPX File" |
| Trails toggle pill | "Trails" (with hiking emoji prefix: "🥾 Trails") |
| Import panel title | "Import GPX File" (unchanged from existing) |
| Import panel description | "Import trail routes and waypoints from AllTrails, Wikiloc, or any GPS app. Tracks appear as map overlays; waypoints become saved spots." |
| Trails list section heading | "Saved Trails" |
| Success message — trail created | "{N} trail{s} added, {M} location{s} added" (extends existing parts.join pattern) |
| Success message — trail only | "{N} trail{s} added" |
| Success message — locations only | "{N} location{s} added" (unchanged) |
| Success message — no data | "No tracks or waypoints found in GPX file" |
| Importing in-progress | "Importing..." (unchanged from existing) |
| Empty state — no trails yet | "No trails imported yet" |
| Error state — import failed | "Failed to import GPX file. Check the file is a valid .gpx and try again." |
| Error state — delete failed | "Could not delete trail. Try again." |
| Delete confirmation | Inline: clicking delete immediately removes the trail with no confirmation dialog — destructive action is low-stakes (trail can be re-imported). Row shows a brief red flash then disappears. |

Source of decisions:
- "Choose GPX File" — standard browser file input label; consistent with existing file input pattern in spots-client.tsx line 453
- "Saved Trails" heading — matches the noun used throughout CONTEXT.md (D-02, D-03)
- No confirmation dialog for delete — trail data can be re-imported from the original GPX; low-stakes enough to skip a modal (consistent with how the app handles other quick destructive actions)
- Import message format — extends existing `parts.join(', ')` pattern (spots-client.tsx lines 136-139)

## Interaction States

### Trails Layer Toggle
| State | Visual |
|-------|--------|
| Active (trails visible) | `bg-stone-800 text-white` / dark: `bg-stone-200 text-stone-900` |
| Inactive (trails hidden) | `bg-stone-200 text-stone-500` / dark: `bg-stone-700 text-stone-400` |
| Hover | `transition-colors` (inherits from existing toggle pattern) |

### File Import Flow
| State | Visual |
|-------|--------|
| Idle | File input visible, no message |
| Importing | Input disabled at `opacity-50`, "Importing..." in `text-amber-600 animate-pulse` |
| Success | Message in `text-emerald-600 dark:text-emerald-400 font-medium` |
| Error | Message in `text-red-600 dark:text-red-400 font-medium` |

### Trail Delete
| State | Visual |
|-------|--------|
| Idle | Trash icon in `text-stone-400` |
| Hover | `text-red-600 dark:text-red-400` |
| Deleting | Icon replaced with spinner `animate-spin h-4 w-4`, row locked |
| Deleted | Row removed from list (no toast, no confirmation — instant) |

### Trail Polyline on Map
| State | Visual |
|-------|--------|
| Default | Solid `#22c55e` green, weight 3, opacity 0.8 |
| Hover | Leaflet default hover (no custom style needed this phase) |
| Popup open | Standard Leaflet popup, styled with inline tokens |
