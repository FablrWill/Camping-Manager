  <action>
1. Add `shareToken String? @unique` to the Trip model in prisma/schema.prisma (place it after `journalEntryAt`).

2. Create and apply the migration:
   - Run `npx prisma migrate dev --name add-trip-share-token` to create the migration.
   - Run `npx prisma generate` to update the client.

3. Create `app/api/trips/[id]/share/route.ts` with two handlers:

   **POST** — Generate share token:
   - Import `generateSlug` from `@/lib/share-location`.
   - Look up the trip by `params.id`. Return 404 if not found.
   - If trip already has a shareToken, return the existing URL (idempotent).
   - Otherwise call `generateSlug()`, save to trip via `prisma.trip.update({ where: { id }, data: { shareToken } })`.
   - Return `{ shareUrl: '/trips/share/{token}' }` with status 200.

   **DELETE** — Clear share token (unshare):
   - Look up the trip by `params.id`. Return 404 if not found.
   - Update trip: `prisma.trip.update({ where: { id }, data: { shareToken: null } })`.
   - Return `{ unshared: true }` with status 200.

   Both handlers: standard try-catch with console.error + JSON error response pattern.
