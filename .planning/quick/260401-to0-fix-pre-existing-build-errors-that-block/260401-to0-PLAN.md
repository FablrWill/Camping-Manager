# Quick Task 260401-to0: Fix Build Errors

## Plan 01: Convert problematic top-level imports to dynamic imports

**Goal:** Get `npx next build` to complete cleanly by converting top-level imports of packages with ESM/native-binding issues to dynamic imports that only load at runtime.

**Root cause:** Next.js page data collection eagerly evaluates API route modules. Packages like `voyageai` (broken ESM directory imports), `better-sqlite3`/`sqlite-vec` (native bindings), `nodemailer`, and `openai` fail during this static analysis phase even though they're only needed at runtime.

**Fix strategy:** Convert top-level `import` statements to dynamic `await import()` calls inside the functions that use them. This defers module loading to runtime when the API is actually called.
