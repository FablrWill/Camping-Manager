2. Create lib/trip-intelligence.ts:
- Export TripIntelligenceReport interface matching spec:
  ```typescript
  interface TripIntelligenceReport {
    stats: { tripCount: number; avgNights: number; totalNights: number; topSeason: string; avgDriveMiles: number }
    patterns: string[]
    gearInsights: { name: string; insight: string; action: 'keep' | 'reconsider' | 'always_bring' }[]
    spotsToRevisit: { name: string; reason: string }[]
    generatedAt: string
  }
  ```
- Export async function generateTripIntelligence(): Promise<TripIntelligenceReport>
- Query all trips with includes: { location: true, packingItems: { include: { gear: true } }, tripFeedbacks: true, mealPlan: true }
- Compute stats locally:
  - tripCount: trips.length
  - For each trip: nights = differenceInDays(endDate, startDate)
  - avgNights: avg of nights, totalNights: sum
  - topSeason: derive season from startDate month (Mar-May=spring, Jun-Aug=summer, Sep-Nov=fall, Dec-Feb=winter), count, pick max
  - avgDriveMiles: for trips with location lat/lon, compute haversine distance from Asheville (35.5951, -82.5515) in miles, average them
- Build a text summary of the data to send to Claude: trip list with dates/locations/duration, gear usage counts from PackingItem usageStatus, TripFeedback summaries, seasonal distribution
- Call anthropic.messages.create with model "claude-sonnet-4-6", system prompt: "You are a camping analytics expert. Analyze this camper's trip history and provide insights. Return JSON matching the schema exactly." Pass the data summary as user message. Request JSON output.
- Parse response with parseClaudeJSON using a Zod schema (add TripIntelligenceReportSchema to lib/parse-claude.ts)
- Fill in stats from local computation (not from Claude), patterns/gearInsights/spotsToRevisit from Claude response
- Return the merged TripIntelligenceReport with generatedAt: new Date().toISOString()

Export the schema and its inferred type.

4. Create app/api/trips/intelligence/route.ts:
- GET handler: prisma.campingProfile.findUnique({ where: { id: 'singleton' } }). If found, return JSON.parse(reportJson). If not found, return { report: null }.
- POST handler: Check trip count first — if < 3, return { error: 'Not enough trips yet. You need at least 3 trips for a camping profile.' } with status 400. Otherwise call generateTripIntelligence(), upsert CampingProfile singleton with JSON.stringify(report) as reportJson and new Date() as generatedAt. Return the report.
- Standard try-catch error handling per project convention.
